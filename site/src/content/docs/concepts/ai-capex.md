# 算力资本开支

AI CapEx 不能只看云厂商花了多少钱，还要拆成“订单、交付、上电、在线容量、折旧和收入兑现”几个阶段。

## 分析框架

| 阶段 | 需要观察的信号 |
| --- | --- |
| 预算 | 云厂商资本开支指引、Neocloud 融资、长期租约。 |
| 订单 | GPU、ASIC、HBM、服务器和网络设备订单。 |
| 交付 | 芯片、整机、机柜、液冷和电源交付节奏。 |
| 上电 | 数据中心接电、能源合同、建设许可证。 |
| 使用 | 训练、推理、API 调用、企业客户迁移。 |
| 财务 | 折旧周期、租赁成本、毛利率和自由现金流。 |

## 为什么适合跨层展示

CapEx 主题天然连接 L1 能源、L2 芯片、L3 基建、L4 模型和 L5 应用。它适合做成网站里的“跨层方法论”专题，而不是只放在单个技术层级下。
