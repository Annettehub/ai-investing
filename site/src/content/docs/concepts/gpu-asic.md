# GPU vs ASIC

GPU 与 ASIC 的竞争不是简单替代关系，而是训练、推理、软件生态、供应链能力和云厂商自研战略之间的再分配。

## 核心判断

| 维度 | GPU | ASIC |
| --- | --- | --- |
| 训练 | 生态、灵活性和互联能力强，短期仍是主导。 | 架构变化快，定制芯片追赶难度高。 |
| 推理 | 通用性强，但成本和能效压力更明显。 | 更适合固定模型、大规模推理和云厂商内部负载。 |
| 软件生态 | CUDA、NVLink、系统软件形成高迁移成本。 | 需要客户有足够规模和工程能力摊薄开发成本。 |
| 供应链 | 依赖先进制程、HBM、先进封装和整机交付。 | 同样依赖台积电、封装和设计服务生态。 |

## 跟踪问题

- 推理需求是否持续高于训练需求。
- Broadcom、Marvell 等 ASIC 设计服务商的订单持续性。
- 云厂商自研芯片是否真的降低总拥有成本。
- NVIDIA 是否通过 Rubin、NVLink 和软件栈继续扩大系统级优势。
