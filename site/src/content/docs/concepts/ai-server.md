# AI 服务器供应链

AI 服务器供应链适合从需求端向上游拆解：模型和应用消耗算力，云厂商扩建数据中心，再传导到芯片、内存、网络、光模块、电源、被动元件和能源。

## 五层映射

| 层级 | 典型环节 |
| --- | --- |
| 应用与模型 | OpenAI、Anthropic、Meta、xAI、字节、阿里等需求方。 |
| 基础设施 | 数据中心、服务器、交换机、光模块、液冷、电源。 |
| 芯片与部件 | GPU、ASIC、HBM、CPU、MLCC、PCB。 |
| 能源 | 电网、发电、储能、数据中心接入能力。 |
| 资本 | 云厂商 CapEx、租赁云、Neocloud 融资和折旧周期。 |

## 跟踪问题

- 训练和推理的需求比例是否继续变化。
- 算力租金、GPU 交付周期和数据中心上电节奏。
- 光模块、HBM、MLCC、PCB 等环节是否出现真实短缺。
- CapEx 增长是否转化为线上可用算力，而不只是订单。
