# Obsidian 可视化知识库配置指南

> 本指南帮助你把 `ai-investing/` 知识库在 Obsidian 中配置为**自动同步 + 可视化图谱**的工作流。
> 配置完成后：WorkBuddy 每次更新知识库 → Obsidian 自动看到变化 → Graph View 实时渲染关联图谱

---

## 第一步：安装 Git 插件（1 分钟）

Obsidian 本身不带 Git 同步功能，需要安装社区插件 `Obsidian Git`。

### 1.1 关闭安全模式

在 Obsidian 中：
1. 点击左下角 **⚙️ 设置**（齿轮图标）
2. 左侧选择 **第三方插件**
3. 关闭右上角的 **安全模式**（滑块向左滑动）
4. 点击 **关闭** 确认

### 1.2 安装 Git 插件

1. 在第三方插件页面，点击 **浏览社区插件**
2. 搜索框输入 `Git`
3. 找到 **Obsidian Git**（作者：Vinzent，下载量最高）
4. 点击 **安装** → 等待安装完成 → 点击 **启用**

### 1.3 验证插件已启用

回到第三方插件列表，确认 `Obsidian Git` 开关为 **蓝色（启用）**。

---

## 第二步：配置自动同步（已预配置，只需确认）

Git 插件的配置文件已预先放置，安装后自动生效：

| 配置项 | 设置值 | 说明 |
|--------|--------|------|
| 自动提交 | ✅ 开启 | 每 30 分钟自动 commit |
| 自动推送 | ✅ 开启 | commit 后自动 push 到 GitHub |
| 提交信息 | `vault backup: {{date}}` | 自动带上时间戳 |
| 自动拉取 | ✅ 开启 | 启动时自动 pull 最新内容 |
| 显示状态栏 | ✅ 开启 | 底部状态栏显示 Git 状态 |

### 如需手动调整

1. 打开设置 → 左侧找到 **Obsidian Git**
2. 展开 **自动提交与推送** 区域
3. 确认以下选项已勾选：
   - [x] 自动提交所有更改（Auto commit all changes）
   - [x] 自动推送（Auto push）
   - [x] 自动拉取（Auto pull）
   - [x] 启动时自动拉取（Auto pull on startup）
4. 提交间隔：`30` 分钟

---

## 第三步：打开 Graph View 知识图谱

### 3.1 打开图谱

1. 点击左侧边栏的 **🔵 图谱** 图标（或按 `Ctrl+G`）
2. 你会看到所有 Markdown 文件以节点形式呈现
3. `[[...]]` 双向链接会自动渲染为连线

### 3.2 图谱已预配置颜色分组

打开图谱后，不同模块的笔记会以不同颜色显示：

| 颜色 | 模块 | 说明 |
|------|------|------|
| 🔵 蓝色 | `entities/` | 公司/标的档案 |
| 🟢 绿色 | `concepts/` | 概念/行业框架 |
| 🟣 紫色 | `hypotheses/` | 假设判断层 |
| ⚪ 灰色 | `sources/` | 来源摘要 |
| 🟠 橙色 | `commands/` | 系统命令 |
| 🔴 粉色 | `output/` | 输出产物 |
| 🩵 青色 | `system/` | 系统协议 |

### 3.3 调整图谱布局

- **滚轮缩放** — 放大/缩小视图
- **拖拽节点** — 手动调整布局
- **点击节点** — 跳转到对应笔记
- **右键空白处** → **恢复默认布局** — 重置

## 第四步：日常使用工作流

### 场景 1：WorkBuddy 更新后，Obsidian 查看最新内容

1. WorkBuddy 完成更新 → 文件自动写入 `ai-investing/` 目录
2. Obsidian Git 插件在 **30 分钟内** 自动 commit + push
3. 你在 Obsidian 中直接看到更新内容（实时，无需刷新）

### 场景 2：在 Obsidian 中手动编辑笔记

1. 在 Obsidian 中编辑任意 `.md` 文件
2. 保存后（`Ctrl+S`），Git 插件会自动检测到变更
3. 30 分钟内自动 commit + push
4. WorkBuddy 下次运行时，可以通过 Git 看到这些变更

### 场景 3：多设备同步

如果你有另一台电脑/笔记本：
1. 在新设备上 clone 同一个 GitHub 仓库
2. 用 Obsidian 打开
3. 启动时自动 pull 最新内容
4. 任何设备的修改都会同步到所有设备

## 第五步：高级技巧

### 5.1 快速在 Graph View 中定位核心标的

在 Graph View 的 **搜索框** 中输入：
- `entities/` — 只看标的档案
- `concepts/` — 只看概念框架
- `H1.1 OR H2.2 OR H4.1` — 只看特定假设
- `YMTC OR 台积电 OR 中芯` — 只看特定公司

### 5.2 用双向链接快速跳转

在任何笔记中点击 `[[entities/YMTC.md]]` 这样的链接，可以直接跳转到对应笔记。

你也可以在笔记中**新建链接**：输入 `[[`，Obsidian 会自动提示已有笔记。

### 5.3 本地图片查看

Obsidian 原生支持 Markdown 图片语法 `![[图片名.png]]`。如果 WorkBuddy 生成的笔记中包含图片，在 Obsidian 中可直接查看。

## 故障排除

| 问题 | 解决方式 |
|------|---------|
| Git 插件显示 "Git not found" | 确保系统已安装 Git（你已经在用 Git，应该没问题） |
| 自动推送失败 | 检查 GitHub 凭据是否已配置（`git push` 是否能手动成功） |
| Graph View 看不到连线 | 确保笔记中使用了 `[[笔记名]]` 格式的双向链接 |
| 颜色分组未生效 | 在 Graph View 中点击 **⚙️ 显示设置** → 确认颜色分组已启用 |
| Obsidian 卡顿 | Graph View 节点过多时可能卡顿，可在搜索框过滤缩小范围 |
