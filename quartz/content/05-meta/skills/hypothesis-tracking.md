# 假设跟踪技能（hypothesis-tracking）

> 能力说明书：设条件 → 每日扫描 → 周汇总 → 升降 certainty% → 回写假设。

## 执行流程

1. **设假设**：每个假设必须可证伪，包含具体数据锚点和证伪条件
2. **设触发条件**：什么数据/事件出现会改变 certainty%
3. **每日扫描 /today**：扫描外部信号，判断是否触发条件
4. **周汇总 /weekly**：汇总假设变化，分析事实驱动 vs 情绪驱动
5. **升降 certainty%**：根据证据强度调整，诚实记录原因
6. **回写假设**：在假设文件末尾追加更新记录
7. **证伪处理**：假设被推翻 → 移入 `00-system/false-beliefs.md`

## 假设文件格式

```markdown
# H1: [假设标题]

## 陈述
[具体可观察的陈述]

## 当前 certainty%
[百分比]

## 证伪条件
[什么数据/事件会证明我错了]

## 触发条件
- [条件1] → +5% / -5%
- [条件2] → +10% / -10%

## 更新记录
- YYYY-MM-DD: 65% → 70%, 原因: ...

## 相关城墙
- W1

## 相关来源
- [[SRC-XXX]]
```
