# 摩擦日志（friction-log）

> 记录 AI 协作中的不爽、错误、低效点，是系统自我进化的燃料。

## 格式

```markdown
- YYYY-MM-DD HH:MM [类型] [场景] [具体问题] [期望行为] [改进建议] [状态]
```

类型：理解错误 / 输出格式不对 / 漏读文件 / 重复劳动 / 工具失效 / 其他

状态：待处理 / 已修复 / 已忽略

## 记录

- 2026-07-04 14:52 [工具失效] [闭环测试执行中] [API 配额耗尽中断执行] [应能继续从中断点恢复] [已通过 active-context.md 断点续传恢复] [已修复]
- 2026-07-04 14:30 [理解错误] [规则晋升流程] [初始设计未明确候选规则需要几次验证才能晋升] [应明确≥3次] [已在 rules.md 中补充晋升流程] [已修复]

## 使用方式

1. 每次发现不爽立即记录
2. `/weekly` 自动回顾 friction-log
3. `/update` 基于 friction-log 提出系统改进建议
4. 修复后更新状态为「已修复」
