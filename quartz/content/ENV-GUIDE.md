# 三环境使用指南

> 白天公司 + 晚上家里 + 手机随时。GitHub 是中枢，WorkBuddy 是入口。

---

## 环境概览

| 环境 | 时间 | 设备 | 可用工具 | 能做什么 |
|------|------|------|---------|---------|
| 🏢 公司 | 白天 | 公司电脑 | WorkBuddy 网页版 + GitHub + 腾讯云 | 查知识库、手动入料、轻量研究 |
| 🏠 家里 | 晚上/周末 | 个人电脑 | WorkBuddy 桌面版 + 全部工具 | 深度研究、配置、批量处理 |
| 📱 手机 | 全天 | 手机 | WorkBuddy 移动端 + 飞书 + 微信读书 | 碎片阅读、快速入料、看报告 |

---

## 核心设计：GitHub 做中枢

```
               GitHub (Annettehub/ai-investing)
              ┌─────────────────────────────────┐
              │  02-kb/  00-system/  03-raw/    │
              │  active-context.md              │
              └───────────┬─────────────────────┘
                          │
        ┌─────────────────┼─────────────────┐
        ▼                 ▼                  ▼
   🏢 公司网页版      🏠 家里桌面版      🤖 GitHub Actions
   只读为主          读写全能            自动飞书同步
   手动入料            深度研究            每天 6 次
```

**原则：** 所有改动最终都落到 GitHub 仓库里。你是单人使用，不存在冲突。

## 🏢 公司环境（白天）

### 限制
- 不能安装桌面软件
- 只能用 WorkBuddy 网页版
- 工具：GitHub ✅ / 腾讯云 ✅ / WebSearch ✅

### 推荐操作

**1. 每日检查（最常用）**
```
"执行 /today 命令检查 00981"
```
→ WorkBuddy 会读取 GitHub 仓库的 active-context + snapshot，搜索最新信息，更新假设概率。

**2. 手动入料（看到好文章时）**
```
直接把文章内容粘贴给 WorkBuddy，说：
"入库这篇关于 XX 的文章到 03-raw/manual/，关联 00981"
```
→ 不需要手动建文件，AI 会创建并提交到 GitHub。

**3. 快速查询**
```
"00981 现在有哪些活跃假设？"
"光刻胶断供最近有什么新进展？"
```

**4. 微信读书笔记入料**
手机截屏/复制笔记 → 粘贴给网页版 WorkBuddy → AI 结构化入库。

### GitHub 操作
- 公司电脑可访问 github.com → 可以直接看 repo 里的文件
- WorkBuddy 网页版通过 GitHub connector 读写文件
- 所有改动会自动 commit 到仓库

## 🏠 家里环境（晚上/周末）

### 全部能力
- WorkBuddy 桌面版（功能最全）
- 本地文件系统访问
- 飞书客户端
- 微信读书桌面版

### 推荐操作

**1. 深度研究（大活）**
```
"执行 /research 研究 TSMC"
```
→ 7步完整研究流程，生成假设和城墙，写入知识库。

**2. 飞书同步（GitHub Actions 自动运行，也可手动触发）**
```
到 GitHub repo → Actions → Sync Feishu Drive → Run workflow
```

**3. 周度复盘（周六）**
```
"执行 /weekly 命令"
```

**4. 配置维护**
- 新增管道连接
- 更新 rules.md
- 清理僵尸假设

**5. 批量处理**
```
"处理 03-raw/ 里所有未入库的文件"
```

## 📱 手机端（全天碎片时间）

### 场景
- 通勤时刷到一篇好文章
- 飞书上收到同事分享的研报
- 微信读书看到关键段落

### 推荐操作

**1. 快速入料（最常用）**
```
在 WorkBuddy 手机端粘贴内容：
"这篇文章讲的是 XX 行业趋势，帮我入库"
```

**2. 随手记录想法**
```
"记录一个想法：NVDA 的护城河可能不是 CUDA 而是..."
```
→ AI 存入 hypothesis/ 或 inbox/。

**3. 查看状态**
```
"我今天需要关注什么？"
"W1 光刻胶断供最新状态是什么？"
```

**4. 微信读书笔记**
```
微信读书 → 笔记 → 复制 → 粘贴到 WorkBuddy：
"这些是我在《芯片战争》里的笔记，帮我整理入库"
```

## 注意事项

1. **GitHub 是唯一真相源** — 不管从哪个环境操作，结果都落在 GitHub
2. **公司环境以读为主** — 深度研究留到家里桌面版
3. **手机以收集为主** — 碎片信息快速捕捉，不要求完美格式
4. **飞书自动化不需要管** — GitHub Actions 每 6 小时自动跑一次
5. **active-context.md 保证断点续传** — 换环境、换设备都不丢上下文
