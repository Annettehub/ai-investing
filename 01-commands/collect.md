# /collect 多源采集命令

## 使用场景
聚合多个信息源（Gmail/飞书/GitHub/微信读书/Web）的新内容，批量导入原料层。

## 完整 Prompt（复制以下全部给 AI）

你正在执行 /collect 多源采集命令。

### 目标
扫描配置的信息源，提取新内容，写入 `03-raw/`。

### 执行步骤

**Step 1: GitHub Stars 检查**
- 获取最近 Star 的项目
- 检查是否有新 Release/重要更新
- 提取项目摘要（一句话 + 为什么关注）
- 输出：`03-raw/github/YYYY-MM-DD-项目名.md`

**Step 2: Gmail 检查（如已连接）**
- 检查 Industry-Intel 标签下未读邮件
- 提取发件人、主题、核心信息
- 输出：`03-raw/gmail/YYYY-MM-DD-主题.md`

**Step 3: WebSearch 行业扫描（可选）**
- 对关注的行业/标的执行一次搜索
- 提取 2-3 条最重要的新闻
- 输出：`03-raw/web/YYYY-MM-DD-行业.md`

**Step 4: 微信读书检查（如有新导出）**
- 扫描 `03-raw/weread/` 中的新文件
- 如有，建议执行 /ingest

**Step 5: 汇总报告**
- 列出本次采集到的所有新内容
- 标记需要立即处理的高优先级项目
- 建议下一步行动（/ingest /research /today）

### 检查清单
- [ ] 不重复已处理的内容
- [ ] 高优先级项目已标记
- [ ] 每条有来源和日期

### 输出格式
```markdown
# [日期] 多源采集报告

## 本次新内容

| 来源 | 标题 | 优先级 | 建议动作 |
|------|------|--------|---------|
| GitHub | ... | 高/中/低 | /ingest |
| Gmail | ... | 高/中/低 | /research |
| Web | ... | 高/中/低 | /today |

## GitHub Stars 检查
[项目名] — [一句话描述] — [上次更新] — [为什么关注]

## Gmail 头版摘要
[发件人] — [主题] — [核心信息一句话]

## 行业信号扫描
[搜索关键词] → [发现]

## 建议下一步
1. 高优先级 → /ingest 或 /research
2. 中优先级 → 纳入 /today 仪表盘
3. 低优先级 → 记录归档

```
