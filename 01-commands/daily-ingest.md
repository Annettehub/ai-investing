## 一、每日固定流程（晚上 9 点执行）

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 扔（白天随时）                                      │
│  → 资料扔进 IMA "Workbuddy同步库"                           │
│  → 或扔进 03-raw/wechat/（微信/网页/手动资料统一入口）      │
├─────────────────────────────────────────────────────────────┤
│  Step 2: 拉（晚上 9 点）                                     │
│  → 对 WorkBuddy 说："跑 IMA 同步"                           │
│  → AI 从 IMA 拉取新增资料 → 03-raw/wechat/                  │
├─────────────────────────────────────────────────────────────┤
│  Step 3: 筛（用户做，AI 不做）                               │
│  → 快速扫 wechat 新文件，删掉明显无关的                     │
│  → 原则：进入 03-raw 的 = 已判定有用，AI 不二次筛选         │
├─────────────────────────────────────────────────────────────┤
│  Step 4: 分类（AI 自动）                                     │
│  → 飞书 → 03-raw/feishu/                                    │
│  → 微信/网页/IMA → 03-raw/wechat/                           │
│  → 播客 → 03-raw/podwise/                                   │
│  → 微信读书 → 03-raw/weread/                                │
│  → 知识星球 → 03-raw/zsxq/                                  │
│  → Gmail → 03-raw/gmail/（待接入）                          │
├─────────────────────────────────────────────────────────────┤
│  Step 5: 摘要（AI 自动生成）                                 │
│  → 每篇生成结构化摘要，存入 02-kb/sources/                  │
│  → 标记关联 entities / concepts / hypotheses                │
├─────────────────────────────────────────────────────────────┤
│  Step 6: 分发确认（用户确认，AI 执行）                       │
│  → AI 提出关联建议清单                                       │
│  → 用户说："全部分发" / "只分发 X" / "修改后再执行"         │
│  → AI 把核心观点写入对应 entity/concept 文件                │
├─────────────────────────────────────────────────────────────┤
│  Step 7: 更新索引（AI 自动）                                 │
│  → 更新 02-kb/index.md                                      │
│  → 追加 02-kb/log.md                                        │
│  → 如有重要发现，更新 active-context.md                     │
└─────────────────────────────────────────────────────────────┘
```

## 二、命名规范

### 进入 wechat 统一入口时的文件名

```
YYYYMMDD-类型-关键词-来源.扩展名
```

| 类型标记 | 含义 |
|---------|------|
| `PDF` | 券商研报、公司财报、行业深度 |
| `微信` | 公众号文章 |
| `研报` | 卖方研究报告 |
| `纪要` | 会议纪要、调研纪要 |
| `新闻` | 行业新闻、公司公告 |
| `博客` | 个人博客、Substack |
| `笔记` | 手动整理的笔记 |

**例子：**
- `20260705-PDF-Serenity-AI芯片评级.pdf`
- `20260705-微信-机器人减速器-饭统戴老板.md`
- `20260705-研报-MLCC涨价-中信证券.pdf`

## 三、标准化检查清单（每篇资料必过）

### 3.1 摘要生成检查项

- [ ] **元信息完整**：来源/日期/类型/可信度/原文链接
- [ ] **核心观点 3-5 条**：每条有证据支撑
- [ ] **关键数据表格**：有出处的量化数据
- [ ] **关联标记**：entities / concepts / hypotheses
- [ ] **待核实项**：标记不确定的信息
- [ ] **质量评估**：信息密度/偏见风险/是否保存

### 3.2 分发决策检查项

| 检查项 | 是 → 去哪 | 否 → 留下 |
|--------|----------|----------|
| 提到具体公司？ | entities/<公司>.md | — |
| 提到跨公司概念？ | concepts/<概念>.md | — |
| 支持/挑战假设？ | hypotheses/<编号>.md | — |
| 都不匹配？ | — | 留在 sources/，等下次 research |

### 3.3 硬规则（不可违反）

1. **不强行关联**：如果暂时找不到归宿，就留在 sources/
2. **不重复复制**：同一份资料的观点拆到不同卡片，不整份复制多次
3. **只存结论**：02-kb 只存"消化后的结论"，原始文件在 03-raw/
4. **来源必标注**：所有数据/观点必须标注出处

## 五、自动化设置（可选）

### 方案 A：全自动（资料量大时）

```

时间: 每天 21:00
动作:
  1. 从 IMA Workbuddy同步库拉取新增资料

  5. 推送摘要清单给用户，等待分发确认
```

### 方案 B：半自动（当前使用）

```
不设置定时任务
用户想跑时说："跑 IMA 同步"
AI 即时执行 Step 2-5
```

## 六、异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| IMA 同步库无新内容 | 告知用户"今日无新增资料" |
| 资料格式无法读取（纯图片） | 尝试 OCR，告知用户"图片内容可能丢失格式" |
| 关联建议不明确 | 标记为"待研究"，不硬关联 |
| 用户未确认分发 | 摘要留在 sources/，index 已更新，下次再议 |
