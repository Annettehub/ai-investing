# /track 跟踪研究命令 — Stage 6: Track（持续跟踪）

> 最后更新: 2026-07-04
> 对应工作流位置: Stage 3 (Research) → Stage 4 (Report) → Stage 5 (Decision) → **Stage 6 (Track)**

## 核心定位

**持续闭环**: 新信号 → **7维度逐层扫描** → 更新确定性 → 影响决策

```
买入 → 进入跟踪循环 → 每次新信号触发扫描 → 判断是否更新confidence% → 影响下一步决策
```

> **买入是进入跟踪循环的起点，不是研究的终点。**

---

## 触发条件

| 类型 | 频率 | 说明 |
|------|------|------|
| **📅 定期** | 每日/每周 `/today` 或 `/weekly` | 自动扫描全部活跃 entity 的假设雷达 |
| **⚡ 事件驱动** | 新数据发布 / 新闻 / 知识星球新文章 | 只扫描受影响的 entity + 关联假设 |
| **💰 价格异动** | 标的涨跌超阈值 | 扫描该标的的全部关联假设 |
| **🔍 手动** | 用户要求 `/track {ticker}` | 对指定标的做全面假设雷达刷新 |

## 跟踪流程

### Step 1：识别新信号

```
今日新信号来源：
□ 03-raw/ 有新文件入库？
□ 知识星球有新文章？
□ 财报/法说会发布？
□ 行业数据更新？
□ 价格异动（涨跌 > X%）？
□ 用户主动提供资料？

如无任何新信号 → 输出"无变化"并跳至结束
```

### Step 2：加载目标 Entity 的假设雷达

读取 `02-kb/entities/{ticker}.md` 的「假设雷达」段：

```markdown
## 假设雷达（7维映射）
| 维度 | 相关假设 | 本标的与假设的关系 |
| D1-Trend | H1.x (XX%) | 一句话关系 |
| ... | ... | ... |
```

**这是每次跟踪的起点** — 你已经知道这个标的与哪些假设有关联，不需要从头扫描。

### Step 3：逐层扫描（核心操作）

对假设雷达中的**每个相关假设**，执行：

```
针对 H{编号} [{标题}] — 当前 certainty% XX%

1. 有新证据吗？
   □ 无 → 保持不变，标记 "stale if 连续N次"
   □ 有 ↓ 继续

2. 新证据是什么？→ 追加到 H*.md 证据时间线
   [来源标签] 具体内容... 🟢支持/🔴削弱/🟡中性

3. Kill Metric 接近触发了吗？
   □ 否 → 正常
   □ ⚠️接近 → 标注警告
   □ ❌触发 → certainty% 归零，移入 false-beliefs

4. Certainty% 要调整吗？（参考变更规则）
   □ 不变
   □ ↑ +X%（理由：___）
   □ ↓ -X%（理由：___）
```

**变更规则**:
- 单个弱证据(🟡) → 不变或 ±5%
- 多个一致中等证据(🟢🟢) → +10~15%
- 单个强反方证据(🔴) → -10~20%
- Kill Metric 接近 → -10% + 加急标记
- 连续3次无变化且无新证据 → 标注"stale"，考虑降级关注度

### Step 4：Kill Thesis 月度自问

每条被扫描的假设，每月至少回答一次：
1. 当前证据是否仍支持假设方向？
2. 是否有新反方证据被忽视？
3. Kill Metric 阈值是否需要收紧/放宽？

### Step 5：输出 & 回写

#### 输出模板（追加到 `04-output/today-YYYY-MM-DD.md`）

```markdown
## 假设跟踪 ({日期})

### {Ticker} {公司名}
| 假设 | 上次 | 本次 | 变化 | 触发原因 |
|------|------|------|:---:|---------|
| H4.1 | 55% | 60% | ↑+5 | [来源] xxx |
| H7.2 | 60% | 62% | ↑+2 | [来源] yyy |
| H6.1 | 50% | 50% | → | 无新证据 |

### 全局汇总
今日扫描 N 个 entity, M 条假设
变动: ↑X条 / ↓Y条 / →Z条 / ⚠️K条接近触发
```

#### 回写清单

| # | 操作 | 目标文件 | 触发条件 |
|---|------|---------|---------|
| 1 | 追加证据行 | `H*.md` evidence timeline | 有新证据 |
| 2 | 更新 certainty% | `H*.md` | 有充分依据 |
| 3 | 更新假设雷达 | `entities/{ticker}.md` | 关系发生变化 |
| 4 | 追加 log.md | `log.md` | 每次 track 必须 |
| 5 | 更新 snapshot.md | `snapshot.md` | confidence% 发生变更 |

## 与 Stage 3 Research 的边界

| | Stage 3 `/research` | Stage 6 `/track` |
|--|---------------------|-------------------|
| 目标 | **建档案** | **盯变化** |
| 7维度角色 | 映射（哪些假设可能相关） | **逐层扫描**（是否需要调confidence%） |
| 假设雷达 | **创建**（首次标记） | **刷新**（判断是否更新） |
| 输出位置 | `entities/{ticker}.md` | `H*.md` + `today-*.md` |
| 频率 | 按需（新标的/大事件） | **定期+事件驱动** |
| 时长 | 可能较长（深度收集信息） | **短平快**（聚焦变化量） |
