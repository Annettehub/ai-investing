# /today 每日跟踪命令

## 使用场景

每日开盘前/收盘后执行，扫描所有活跃假设和城墙的关键信号，输出一页纸日报。

## 发给 AI 的完整 Prompt

你正在执行 /today 每日信号扫描命令。

### 目标

1. 扫描所有活跃假设 H1-H10 的触发条件
2. 检查城墙 W1-W4 的预警信号是否出现
3. 输出一页纸日报，包含：假设 certainty% 变化、新信号、风险预警、今日动作

### 输入

1. 当前 `02-kb/hypotheses/` 下所有活跃假设（读取）
2. 当前 `02-kb/walls/` 下所有活跃城墙（读取）
3. 用户关注池（粘贴或读取 `02-kb/watchlist.md`）
4. 外部数据源（WebSearch / 飞书 / Gmail 等）

### 执行步骤

**Step 1: 读取假设与城墙**
- 列出每个假设的当前 certainty%、触发条件、证伪条件
- 列出每个城墙的预警信号、对应假设

**Step 2: 外部信号扫描**
- 对关键标的/关键词执行 WebSearch
- 检查 03-raw/ 中是否有新文件（feishu/gmail/wechat/podwise/weread/zsxq）
- 读取新闻摘要，判断是否与假设/城墙相关

**Step 3: 信号分类**
- 🔴 Jalapeño：可能改变决策，立即汇报
- 🟡 跟踪：影响假设 certainty%，纳入日报
- 🟢 噪音：记录但不行动
- ⚪ 待核实：信息不完整，标记 [待核实]

**Step 4: 更新假设 certainty%**
- 对新信号，判断支持/挑战哪个假设
- 根据证据强度调整 certainty%（±5% / ±10% / ±15%）
- 在假设文件中追加更新记录

**Step 5: 生成日报**

### 输出格式

```markdown
# YYYY-MM-DD 每日信号扫描

## 一、假设状态总览
| 假设 | 原 certainty% | 新 certainty% | 变化原因 | 信号来源 |
|------|--------------|---------------|---------|---------|
| H1 | 65% | 70% | 村田涨价新闻 | SRC-XXX |

## 二、城墙预警
| 城墙 | 状态 | 预警信号 | 是否触发 | 建议 |
|------|------|---------|---------|------|
| W1 | 休眠 | 光刻胶断供 | 否 | 继续观察 |

## 三、新信号摘要
1. [信号] → [关联假设/城墙] → [优先级]

## 四、今日动作
- [ ] 高优先级：...
- [ ] 中优先级：纳入 /weekly
- [ ] 低优先级：归档

## 五、回写记录
- 已更新 H1/H2/... 的 certainty%
- 已归档信号到 02-kb/sources/（如有）
```

### 检查清单
- [ ] 每个假设都有当前 certainty%
- [ ] 每个信号都标注来源和日期
- [ ] 重大信号已立即向用户汇报
- [ ] 假设文件已追加更新记录
- [ ] 日报已保存到 `04-output/today/YYYY-MM-DD.md`
