# 价值投资知识库框架

## 定位

价值投资子系统用于沉淀长期有效的企业判断、风险判断和技术变迁判断框架。它不直接生成买卖建议，也不替代 AI 产业投研中的行业跟踪；它负责回答：

> 什么样的企业值得长期研究？在不确定性中如何避免永久性损失？技术变化后，价值投资原则如何继续有效？

## 三个长期话题

| 话题 | 层级 | 核心问题 | 主要输出 |
|---|---|---|---|
| [01 巴芒的价值投资理念](topics/01-巴芒的价值投资理念.md) | 原则层 | 什么是好生意，什么价格值得买，如何长期持有？ | 原则卡、企业判断清单 |
| [02 跨学科思维、风险与周期](topics/02-跨学科思维、风险与周期.md) | 决策层 | 在信息不完整、波动和周期中，如何减少判断错误？ | 思维模型卡、风险清单、周期判断 |
| [03 科技时代的价值投资](topics/03-科技时代的价值投资.md) | 应用层 | 技术快速变化时，如何区分长期价值、周期景气和叙事泡沫？ | 科技企业分析、技术周期案例 |

这三个话题不是三份书单，而是一条判断链：

```text
巴芒原则
  -> 跨学科模型、风险与周期
  -> 科技企业和技术变迁中的具体应用
```

## 与微信读书资料的使用规则

同一本书可以服务多个话题，但只建立一个来源摘要，通过链接复用，不复制全文。

| 资料层 | 放置内容 | 例子 |
|---|---|---|
| 来源 | 书籍、股东信、访谈的消化摘要 | `02-kb/sources/SRC-WEREAD-*.md` |
| 概念 | 从多个来源提炼出的稳定概念 | 能力圈、护城河、反脆弱、报酬递增 |
| 话题 | 把概念组织成一个可阅读、可复盘的判断系统 | 本目录 `topics/` |
| 案例 | 把话题应用到一个企业或技术周期 | `案例卡-台积电.md` |

## 第一轮只处理的核心资料

为了避免广撒网，第一轮只围绕八个来源建立三条主线：

| 话题 | 首批核心来源 | 用途 |
|---|---|---|
| 巴芒理念 | 巴菲特致股东信、穷查理宝典、巴菲特的护城河 | 建立企业质量、护城河、资本配置和长期持有原则 |
| 思维与风险 | 反脆弱、霍华德·马克斯备忘录、复杂经济学 | 建立尾部风险、周期、第二层思维和路径依赖框架 |
| 科技投资 | 泡沫逃生、技术革命与金融资本、技术的本质 | 建立技术周期、创新扩散、泡沫与长期价值判断 |

现有知识库中的 AI、半导体和 NVIDIA 来源暂时只作为第三个话题的应用材料，不立即扩展为更多价值投资子话题。

## 入库门槛

一份新资料只有在能够回答以下至少一个问题时，才进入价值投资子系统：

1. 它改变了对企业价值、护城河或资本配置的理解吗？
2. 它提供了识别风险、周期或错误定价的新模型吗？
3. 它能解释技术变化如何创造或摧毁长期竞争优势吗？

只提供新闻、观点或短期行情的内容继续留在 `03-raw`，不直接升级为价值投资知识。

## 推荐工作顺序

```text
选一个话题
  -> 选 2-3 个核心来源
  -> 提炼 3-5 张概念卡
  -> 形成一个判断清单
  -> 用一个企业或技术周期做案例
  -> 周度复盘：哪些原则有效，哪些需要修正
```
