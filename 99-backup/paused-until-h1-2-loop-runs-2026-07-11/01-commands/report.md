# /report — 全池周报

> 每周六自动运行（或手动触发）。汇总一周持仓表现、观察动态、产业信号。

## 前置条件

- 该周已完成 ≥ 2 次 `/scan`（否则数据不足）
- 读取 `config/watchlist.md` 获取最新股票池
- 读取各 `02-kb/company/*.md` 获取标的背景

## 执行步骤

### Step 1: 汇总本周 /scan 记录
- 读取 `04-output/scan-*.md` 中本周的所有扫描记录
- 提取每只标的的本周涨跌幅、异动次数、关键事件

### Step 2: 持仓周报

对每只持仓标的（6只）：

```
### 09992 泡泡玛特
- 本周涨跌：+X.X%
- 关键事件：[事件1, 事件2]
- 指标更新：[有变化的具体指标]
- 下周关注：[财报日期 / 关键节点 / 风险]
- 操作建议：持有 / 加仓 / 减仓 / 观望
```

### Step 3: 观察池动态

对每只观察标的（5只）：

```
### 00981 中芯国际
- 本周涨跌：+X.X%
- 假设更新：[H1-H6 概率变化]
- 产业信号：[来自 themes/ 的相关事件]
- 是否考虑建仓：信号积累中 / 触发条件未满足 / ...
```

### Step 4: 产业信号汇总
- 按 4 个产业方向，汇总一周的重要事件
- 标注与持仓/观察标的的关联

### Step 5: 仓位诊断
- 组合集中度：重仓标的行业分布
- 风险敞口：[中美关系/汇率/利率/行业政策]
- 建议：[是否需要调整仓位结构]

### Step 6: 下周前瞻
- 下周财报日历
- 关键宏观事件
- 重点标的需要关注的节点

### Step 7: 更新 company 文件
将本周确认的事件写入对应 `02-kb/company/*.md` 的事件日志。

输出到 `04-output/report-YYYY-MM-DD.md`

---

## 用法

```
/report
```
