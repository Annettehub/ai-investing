# /trade 决策命令

## 使用场景
在/research和/today基础上，做出建仓/加仓/减仓/清仓决策。

## 发给Kimi的完整Prompt（复制以下全部）

你正在执行 /trade 决策命令。

### 目标
基于当前研究，给出决策建议，但**最终决策权在人**。

### 输入
1. 标的entity档案（粘贴）
2. 当前假设和certainty%（粘贴）
3. 当前城墙状态（粘贴）
4. 当前估值数据（粘贴）
5. 我的风险偏好和仓位限制

### 执行步骤

**Step 1: 认知层级评估**
- 当前对该标的的认知处于哪一层？
  - L1感知变革 → 0-1%观察仓
  - L2懂产业链 → 1-3%试探仓
  - L3有专家网络 → 3-5%基础仓
  - L4有跟踪纪律 → 5-10%核心仓
  - L5有体感 → 10%+重仓

**Step 2: 假设-仓位映射**
- 核心假设的certainty%加权平均是多少？
- 城墙风险是否已被定价？
- 市场共识 vs 我的非共识差异？

**Step 3: 决策矩阵**

**Step 4: 退出预演**
- 建仓前写下："我会在什么信号出现时减仓/清仓？"
- 格式：`如果[具体信号] → [行动]`

**Step 5: 行动建议**
- 建议仓位（%）
- 建议建仓节奏（一次性/分3批/分5批）
- 关键跟踪指标（建仓后每周必看）

### 检查清单
- [ ] 认知层级评估诚实（不要高估）
- [ ] 决策矩阵覆盖了乐观/基准/悲观三种情景
- [ ] 退出预演有具体信号，不是模糊表述
- [ ] 仓位建议符合认知层级（不要越级）

### 输出格式
```markdown
# [标的] 决策建议

## 认知层级评估
## 假设-仓位映射
## 决策矩阵
## 退出预演
## 行动建议（含仓位和节奏）
## 风险提示
