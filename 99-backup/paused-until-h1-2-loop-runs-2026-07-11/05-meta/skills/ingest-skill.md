# 信息入库技能（ingest-skill）

> 能力说明书：把原始资料转化为结构化知识库条目的标准流程。

## 能力定义

- **输入**：研报/纪要/新闻/播客/读书笔记/邮件的原始文本或 `03-raw/` 文件
- **输出**：`02-kb/sources/` 下的结构化摘要 + 索引更新
- **人工验收点**：是否关联了正确的 entities/concepts/hypotheses

## 执行流程

1. **判断来源**：识别作者、机构、日期、类型、可信度
2. **提取核心观点**：3-5 条，每条附证据
3. **提取关键数据**：整理成表格，标注出处
4. **关联知识库**：
   - entities: [[标的名称]]
   - concepts: [[概念名称]]
   - hypotheses: 支持/挑战 H 编号
5. **标记待核实项**：不确定的事实加 `[待核实]`
6. **质量评估**：信息密度、偏见风险、保存价值
7. **更新索引**：`02-kb/index.md` + `02-kb/sources/_index.md` + `02-kb/log.md`

```markdown
- 关联 concepts:
- 关联 hypotheses:

## 待核实项
- [ ]

- 信息密度：
- 偏见风险：
- 是否值得长期保存：
```

## 停止条件

- [ ] 生成了结构化摘要
- [ ] 至少关联了一个 entity 或 concept
- [ ] 更新了索引文件
- [ ] 用户确认质量 OK
