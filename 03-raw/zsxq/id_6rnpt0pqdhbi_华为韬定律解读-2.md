### **韬定律最大的问题就是，它叫「时间缩微」，但最后仍然回到 MTr/mm²**

华为对韬定律的官方描述，是用时间常数 τ 的缩短来替代传统几何尺寸缩微，并把这套方法延伸到器件、电路、芯片、系统四个层级。其中 LogicFolding 逻辑折叠技术被描述为能缩短关键路径布线、降低互连电阻与寄生电容，从而提升晶体管密度、性能与能效。

这套说法最大的问题在于，它名义上是「时间缩微」，但公开材料里没有给出足以工程验证的时间定义。

如果 τ 是时间，那它到底是哪一个时间？是 transistor RC delay？是 FO4 delay？是 critical path delay？是 SRAM access latency？是 NoC latency？是 CPU pipeline stage time？是 GPU memory fetch time？还是 app end-to-end execution time？

华为官方把 device、circuit、chip、system 都放进同一个 τ 框架里，听起来宏大，但其实把很多不同层级的问题混在了一起。

device 层的 τ 可以是物理 RC，circuit 层可以是 critical path，chip 层变成 workload-driven instruction/data flow，system 层又变成 UnifiedBus 与 memory semantics。

这些东西都跟「时间」有关，但它们不是同一个可以直接相加、比较、预测的物理量。所以韬定律目前更像是一个设计哲学或工程叙事，跟「定律」一毛钱关系也没有。

真正的定律至少要有变量定义、量测方法、缩放关系、预测公式，以及可验证或可证伪的边界。

但从目前公开内容看，它没有给出「τ 怎么算」「不同层级如何归一化」「性能、能效、密度之间如何量化转换」这些核心定义。

更讽刺的是，何庭波论文说韬定律要抛开几何缩微、改谈时间缩微，但最后宣传口径又回到了 MTr/mm²。

大肆宣传 Kirin 2026 被外部整理为 238 MTr/mm²、晶体管密度提升 53.5%，2031 年目标则是 400+ MTr/mm²、等效 1.4nm。

这就是问题所在，既然核心是时间，为什么最后还是拿晶体管密度去对标台积电未来 1.4nm？

如果真的是一条新的「时间缩微定律」，就应该用 delay、latency、energy-delay product、data movement time、system execution time 这些时间相关指标建立可比性。

但现在的结果是，先宣称「不再看几何」，再用自家公式把 MTr/mm² 放大，最后拿这个「等效密度」去对标业界先进制程的真实物理密度。这就是典型的口径切换。

这个现象我没破口大骂已经是足够收敛了。

### **用自定义 MTr 放大值，去对标业界物理 MTr**

我们前面讨论的口径是：155 MTr/mm² + 83 MTr/mm² = 238 MTr/mm²。这是 LogicFolding 对外最容易被传播的数字。但若按照韬公式修正系数 1.357，这三个数字的工程含义会完全不同。

问题非常清楚，如果 238 是经过韬公式放大后的等效 MTr，那它不能和台积电、三星、Intel 那种物理制程口径下的 MTr/mm² 直接对比。

因为台积电 N3、N2、未来 A14 / 1.4nm 的 MTr/mm²，通常是基于实际制程设计规则、标准 cell、SRAM、logic density、CPP、MMP 等物理微缩能力。

而华为这个 238 或1.4nm的 400+，混入了 3D stacking、投影面积折算、架构等效密度、甚至系统层面的资料流收益。这两者不是同一把尺。

因此，把「2031 年达到等效 1.4nm 密度」拿来对标全球先进制程，本质上是断章取义。

这不是「SMIC 2031 年会做出物理 1.4nm」，而是「华为希望用自己的等效密度口径，让系统结果接近 1.4nm 级」。这两件事差很多。

### **韬定律是把业界已有路线重新包装**

韬定律所涵盖的方向，其实都是业界早就在做的内容，缩短互连、降低 RC、3D stacking、先进封装、chiplet / heterogeneous integration、大 cache / system cache、软硬协同、资料流优化、memory hierarchy 重构、压缩与稀疏计算、OS / compiler / runtime 参与调度。

这些方向都有价值，但它们不构成一个全新的半导体物理定律。

TSMC SoIC 本来就支援不同尺寸、不同功能、不同节点的 known-good dies 做高密度垂直整合，目标就是降低 RLC、提高性能、降低功耗。

Samsung 早在 X-Cube 技术展示中，就用 7nm test chip 把 SRAM 叠在 logic die 上，透过 TSV 缩短 die 间路径，提高资料速度与能效。AMD 3D V-Cache 则早已经量产，官方明确列出 64MB L3 cache die、TSV、direct copper-to-copper bond 等技术。

所以严格说，韬定律没有脱离目前半导体行业框架。

它不是一条新路，而是把先进封装、3D SRAM、资料流优化、系统协同这些已存在的路线包装成一套新的叙事。

真正值得讨论的，不是「韬定律是不是取代摩尔定律」，而是在 SMIC 制程落后的情况下，华为能不能用高成本 3D SRAM 和架构协同，把手机 SoC 的有效性能拉回主流区间。

### **Kirin 9030 的现实：SMIC N+3 仍然落后**

Kirin 2026 / 9050 的设计必须从 Kirin 9030 的现实出发。

而Kirin 9030 采用 SMIC N+3 制程，是 N+2 / 7nm-class 的 scaled evolution。虽然 N+3 有密度进步，但在微缩程度上仍然明显落后于台积电、三星的商用 5nm 先进节点。

这意味着华为不可能只靠制程去追 Qualcomm、Apple、MediaTek。

如果 Kirin 9030 约 120 多 mm²，而且下层 155 MTr/mm² 需要除以 1.357，那么它的真实密度大约是 114 MTr/mm²。

这与我们讨论中的 SMIC N+3 真实 MTr 约 110–120 MTr/mm² 是一致的。

因此 Kirin 2026 / 9050 真正能做的，不是让 N+3 突然变成 N3P，而是用非常昂贵、较为激进的封装和 SRAM 设计去补短板。

### **kirin 2026/9050 的重点不是 CPU，而是 GPU / NPU / 资料搬移**

根据华为官方提升图，以 2025 为基准，2026 年 LogicFolding 的提升大约如下。

![](images/ddbcb930049656b2e6b3.png)

这组数字可能有企业自己的口径与优化，但也提供了一个直观可参考的范围。

2026年的CPU 提升相对保守，GPU 提升大，NPU 提升极大。

这说明 Kirin 9050 不是一颗主打CPU性能的芯片。

从这个提升幅度以及华为韬定律论文里释放的资料，笔者猜测它更像是，下层扩 GPU / NPU compute，上层用 3D SRAM 减少资料搬移，再用软体、runtime、driver 做场景化调度的一颗芯片。

GPU +38%、能效 +40% 这个方向，尤其像「增加近端 cache / HPM，减少 LPDDR 访问」。

NPU +140%、能效 +81% 则不可能只靠 SRAM，必须来自 NPU MAC 阵列扩大、低精度、稀疏、operator fusion、activation compression、compiler / runtime 分块，以及上层 SRAM 作 activation staging。

### **Kirin 2026 / 9050的设计可能下层算，上层存**

根据所有官方公开资料，所有条件不断收敛后，我会这样设计 Kirin 2026 / 9050。

下层 compute die 约 120–123 mm²，上层 active SRAM die 约 36–40 mm²。上层不是 logic die，而是 shared 3D SRAM pool。高热 compute 全部留在下层。

**下层 die，跟上一代的9030差不多 120–125 mm²，不缩小。**

我看到有小作文说华为利用logicfolding逻辑折叠可以比上代虽小面积，但是如果要达到华为官方的性能提升水平，那下面这一颗die还是得跟之前的9030接近，而9030比苹果，高通得竞品都来得大一点。

而9050虽然把SRAM移到了上层，但下层的主die还是得保持原本的面积，这也正是GPU能有不错提升，NPU有大提升的核心原因，永远都是面积的问题，区别是你移走了SRAM，多出来的空间可以增加GPU跟NPU。

所以下层的主die，不应该因为 SRAM 搬到上层就缩小到 100mm²或以下。

原因很简单：华为的问题不是成本最小化，而是制程落后。如果把 SRAM 搬走后释放的面积拿来省成本，那性能追不上。如果把释放的面积拿来扩 GPU、NPU、NoC、电源网格，才有可能对应官方图上的 GPU / NPU 提升。

因此笔者根据华为官方公布的提升比例认为 9050 的下层策略应该是：CPU 小改，GPU 做宽，NPU 扩大，频率不硬拉。

![](images/8a12da1ba7cdada3a1df.png)

**上层 die：36–40mm² 的 3D SRAM，而不是越大越好的暴力方案**

上层 83 MTr/mm² 按 1.357 修正后为 61.16 MTr/mm²。如果主要是 6T SRAM，则 61.16 ÷ 6 = 10.19 Mbit/mm²，换成容量约 1.27 MB/mm²。

如果只是要跟上苹果，高通，联发科等国际旗舰手机SoC 的 SRAM 那 10mm² 左右的面积就够了。

但华为官方图的 GPU +38%、NPU +140% 不是小幅 cache 增量能解释的。

反过来，60mm² 上层 die 又太像桌面级 3D cache，不像手机量产 SoC 的保守热设计。

因此最合理的方案是 36–40mm² active SRAM die，约 39–46MB 可用 3D SRAM。

### **上层 SRAM 到底放什么？**

上层不能放 GPU shader，不能放 NPU MAC，不能放 CPU core，也不应该放 GPU L1 / NPU L0 这种最热、最频繁访问的 local SRAM。

![](images/05eba67f4d256fa1032f.png)

 

上层不是 GPU HPM 固定区 + NPU activation 固定区 + ISP 固定区同时满载。上层是一池 shared SRAM，按场景动态分配。

![](images/14493736da28003dac1f.png)

 

对应官方性能图

![](images/2b969ad8dfc88adf9c40.png)

 因此 9050 的真正架构不是「上下层计算」，而是下层算力扩大，上层资料池降低搬运成本。

### **2027 怎么提升？**

2027 的基础假设是：SMIC N+4 真实 MTr 从 N+3 的约 113 提升到约 130。这相当于 130 ÷ 113 - 1 ≈ 15%。

15% 的真实密度提升很重要，但它不能单独解释华为图上 2027 的巨大提升。

![](images/9e07ec2f3727531c0e81.png)

 

所以 2027 不可能只是 N+4 制程升级。合理推演是，N+4 给下层 compute die 约 15% 密度余裕，第二代 LogicFolding 让上层 SRAM 密度同步提高，上层 active die 从 36–40mm² 提高到 40–44mm²，封装热路径更成熟，GPU/NPU 的 dataflow 和 compression 更成熟，HarmonyOS、driver、compiler、NPU runtime 对 3D SRAM 的调度也更成熟。

如果上层 SRAM 密度按 N+4 比例提升，则 61.16 × 130 ÷ 113 ≈ 70.3 MTr/mm²；换算 SRAM 约 70.3 ÷ 6 ÷ 8 ≈ 1.46 MB/mm²。

这样 2027 的提升才比较合理，不是 N+4 单独救场，而是 N+4 + 第二代 3D SRAM + 更成熟资料流协同共同作用。

### **最终定论：韬定律是叙事，3D SRAM 是补课**

### **韬定律最大的问题就是，它叫「时间缩微」，但最后仍然回到 MTr/mm²**

华为对韬定律的官方描述，是用时间常数 τ 的缩短来替代传统几何尺寸缩微，并把这套方法延伸到器件、电路、芯片、系统四个层级。其中 LogicFolding 逻辑折叠技术被描述为能缩短关键路径布线、降低互连电阻与寄生电容，从而提升晶体管密度、性能与能效。

这套说法最大的问题在于，它名义上是「时间缩微」，但公开材料里没有给出足以工程验证的时间定义。

如果 τ 是时间，那它到底是哪一个时间？是 transistor RC delay？是 FO4 delay？是 critical path delay？是 SRAM access latency？是 NoC latency？是 CPU pipeline stage time？是 GPU memory fetch time？还是 app end-to-end execution time？

华为官方把 device、circuit、chip、system 都放进同一个 τ 框架里，听起来宏大，但其实把很多不同层级的问题混在了一起。

device 层的 τ 可以是物理 RC，circuit 层可以是 critical path，chip 层变成 workload-driven instruction/data flow，system 层又变成 UnifiedBus 与 memory semantics。

这些东西都跟「时间」有关，但它们不是同一个可以直接相加、比较、预测的物理量。所以韬定律目前更像是一个设计哲学或工程叙事，跟「定律」一毛钱关系也没有。

真正的定律至少要有变量定义、量测方法、缩放关系、预测公式，以及可验证或可证伪的边界。

但从目前公开内容看，它没有给出「τ 怎么算」「不同层级如何归一化」「性能、能效、密度之间如何量化转换」这些核心定义。

更讽刺的是，何庭波论文说韬定律要抛开几何缩微、改谈时间缩微，但最后宣传口径又回到了 MTr/mm²。

大肆宣传 Kirin 2026 被外部整理为 238 MTr/mm²、晶体管密度提升 53.5%，2031 年目标则是 400+ MTr/mm²、等效 1.4nm。

这就是问题所在，既然核心是时间，为什么最后还是拿晶体管密度去对标台积电未来 1.4nm？

如果真的是一条新的「时间缩微定律」，就应该用 delay、latency、energy-delay product、data movement time、system execution time 这些时间相关指标建立可比性。

但现在的结果是，先宣称「不再看几何」，再用自家公式把 MTr/mm² 放大，最后拿这个「等效密度」去对标业界先进制程的真实物理密度。这就是典型的口径切换。

这个现象我没破口大骂已经是足够收敛了。

### **用自定义 MTr 放大值，去对标业界物理 MTr**

我们前面讨论的口径是：155 MTr/mm² + 83 MTr/mm² = 238 MTr/mm²。这是 LogicFolding 对外最容易被传播的数字。但若按照韬公式修正系数 1.357，这三个数字的工程含义会完全不同。

问题非常清楚，如果 238 是经过韬公式放大后的等效 MTr，那它不能和台积电、三星、Intel 那种物理制程口径下的 MTr/mm² 直接对比。

因为台积电 N3、N2、未来 A14 / 1.4nm 的 MTr/mm²，通常是基于实际制程设计规则、标准 cell、SRAM、logic density、CPP、MMP 等物理微缩能力。

而华为这个 238 或1.4nm的 400+，混入了 3D stacking、投影面积折算、架构等效密度、甚至系统层面的资料流收益。这两者不是同一把尺。

因此，把「2031 年达到等效 1.4nm 密度」拿来对标全球先进制程，本质上是断章取义。

这不是「SMIC 2031 年会做出物理 1.4nm」，而是「华为希望用自己的等效密度口径，让系统结果接近 1.4nm 级」。这两件事差很多。

### **韬定律是把业界已有路线重新包装**

韬定律所涵盖的方向，其实都是业界早就在做的内容，缩短互连、降低 RC、3D stacking、先进封装、chiplet / heterogeneous integration、大 cache / system cache、软硬协同、资料流优化、memory hierarchy 重构、压缩与稀疏计算、OS / compiler / runtime 参与调度。

这些方向都有价值，但它们不构成一个全新的半导体物理定律。

TSMC SoIC 本来就支援不同尺寸、不同功能、不同节点的 known-good dies 做高密度垂直整合，目标就是降低 RLC、提高性能、降低功耗。

Samsung 早在 X-Cube 技术展示中，就用 7nm test chip 把 SRAM 叠在 logic die 上，透过 TSV 缩短 die 间路径，提高资料速度与能效。AMD 3D V-Cache 则早已经量产，官方明确列出 64MB L3 cache die、TSV、direct copper-to-copper bond 等技术。

所以严格说，韬定律没有脱离目前半导体行业框架。

它不是一条新路，而是把先进封装、3D SRAM、资料流优化、系统协同这些已存在的路线包装成一套新的叙事。

真正值得讨论的，不是「韬定律是不是取代摩尔定律」，而是在 SMIC 制程落后的情况下，华为能不能用高成本 3D SRAM 和架构协同，把手机 SoC 的有效性能拉回主流区间。

### **Kirin 9030 的现实：SMIC N+3 仍然落后**

Kirin 2026 / 9050 的设计必须从 Kirin 9030 的现实出发。

而Kirin 9030 采用 SMIC N+3 制程，是 N+2 / 7nm-class 的 scaled evolution。虽然 N+3 有密度进步，但在微缩程度上仍然明显落后于台积电、三星的商用 5nm 先进节点。

这意味着华为不可能只靠制程去追 Qualcomm、Apple、MediaTek。

如果 Kirin 9030 约 120 多 mm²，而且下层 155 MTr/mm² 需要除以 1.357，那么它的真实密度大约是 114 MTr/mm²。

这与我们讨论中的 SMIC N+3 真实 MTr 约 110–120 MTr/mm² 是一致的。

因此 Kirin 2026 / 9050 真正能做的，不是让 N+3 突然变成 N3P，而是用非常昂贵、较为激进的封装和 SRAM 设计去补短板。

### **kirin 2026/9050 的重点不是 CPU，而是 GPU / NPU / 资料搬移**

根据华为官方提升图，以 2025 为基准，2026 年 LogicFolding 的提升大约如下。

![](images/ddbcb930049656b2e6b3.png)

这组数字可能有企业自己的口径与优化，但也提供了一个直观可参考的范围。

2026年的CPU 提升相对保守，GPU 提升大，NPU 提升极大。

这说明 Kirin 9050 不是一颗主打CPU性能的芯片。

从这个提升幅度以及华为韬定律论文里释放的资料，笔者猜测它更像是，下层扩 GPU / NPU compute，上层用 3D SRAM 减少资料搬移，再用软体、runtime、driver 做场景化调度的一颗芯片。

GPU +38%、能效 +40% 这个方向，尤其像「增加近端 cache / HPM，减少 LPDDR 访问」。

NPU +140%、能效 +81% 则不可能只靠 SRAM，必须来自 NPU MAC 阵列扩大、低精度、稀疏、operator fusion、activation compression、compiler / runtime 分块，以及上层 SRAM 作 activation staging。

### **Kirin 2026 / 9050的设计可能下层算，上层存**

根据所有官方公开资料，所有条件不断收敛后，我会这样设计 Kirin 2026 / 9050。

下层 compute die 约 120–123 mm²，上层 active SRAM die 约 36–40 mm²。上层不是 logic die，而是 shared 3D SRAM pool。高热 compute 全部留在下层。

**下层 die，跟上一代的9030差不多 120–125 mm²，不缩小。**

我看到有小作文说华为利用logicfolding逻辑折叠可以比上代虽小面积，但是如果要达到华为官方的性能提升水平，那下面这一颗die还是得跟之前的9030接近，而9030比苹果，高通得竞品都来得大一点。

而9050虽然把SRAM移到了上层，但下层的主die还是得保持原本的面积，这也正是GPU能有不错提升，NPU有大提升的核心原因，永远都是面积的问题，区别是你移走了SRAM，多出来的空间可以增加GPU跟NPU。

所以下层的主die，不应该因为 SRAM 搬到上层就缩小到 100mm²或以下。

原因很简单：华为的问题不是成本最小化，而是制程落后。如果把 SRAM 搬走后释放的面积拿来省成本，那性能追不上。如果把释放的面积拿来扩 GPU、NPU、NoC、电源网格，才有可能对应官方图上的 GPU / NPU 提升。

因此笔者根据华为官方公布的提升比例认为 9050 的下层策略应该是：CPU 小改，GPU 做宽，NPU 扩大，频率不硬拉。

![](images/8a12da1ba7cdada3a1df.png)

**上层 die：36–40mm² 的 3D SRAM，而不是越大越好的暴力方案**

上层 83 MTr/mm² 按 1.357 修正后为 61.16 MTr/mm²。如果主要是 6T SRAM，则 61.16 ÷ 6 = 10.19 Mbit/mm²，换成容量约 1.27 MB/mm²。

如果只是要跟上苹果，高通，联发科等国际旗舰手机SoC 的 SRAM 那 10mm² 左右的面积就够了。

但华为官方图的 GPU +38%、NPU +140% 不是小幅 cache 增量能解释的。

反过来，60mm² 上层 die 又太像桌面级 3D cache，不像手机量产 SoC 的保守热设计。

因此最合理的方案是 36–40mm² active SRAM die，约 39–46MB 可用 3D SRAM。

![](images/0d43758d265eb8d32f50.png)

### **上层 SRAM 到底放什么？**

上层不能放 GPU shader，不能放 NPU MAC，不能放 CPU core，也不应该放 GPU L1 / NPU L0 这种最热、最频繁访问的 local SRAM。

![](images/05eba67f4d256fa1032f.png)

 

上层不是 GPU HPM 固定区 + NPU activation 固定区 + ISP 固定区同时满载。上层是一池 shared SRAM，按场景动态分配。

![](images/14493736da28003dac1f.png)

 

对应官方性能图

![](images/2b969ad8dfc88adf9c40.png)

 因此 9050 的真正架构不是「上下层计算」，而是下层算力扩大，上层资料池降低搬运成本。

### **2027 怎么提升？**

2027 的基础假设是：SMIC N+4 真实 MTr 从 N+3 的约 113 提升到约 130。这相当于 130 ÷ 113 - 1 ≈ 15%。

15% 的真实密度提升很重要，但它不能单独解释华为图上 2027 的巨大提升。

![](images/9e07ec2f3727531c0e81.png)

 

所以 2027 不可能只是 N+4 制程升级。合理推演是，N+4 给下层 compute die 约 15% 密度余裕，第二代 LogicFolding 让上层 SRAM 密度同步提高，上层 active die 从 36–40mm² 提高到 40–44mm²，封装热路径更成熟，GPU/NPU 的 dataflow 和 compression 更成熟，HarmonyOS、driver、compiler、NPU runtime 对 3D SRAM 的调度也更成熟。

如果上层 SRAM 密度按 N+4 比例提升，则 61.16 × 130 ÷ 113 ≈ 70.3 MTr/mm²；换算 SRAM 约 70.3 ÷ 6 ÷ 8 ≈ 1.46 MB/mm²。

这样 2027 的提升才比较合理，不是 N+4 单独救场，而是 N+4 + 第二代 3D SRAM + 更成熟资料流协同共同作用。

### **最终定论：韬定律是叙事，3D SRAM 是补课**

韬定律的时间概念本身就是一个幌子，核心内容只是一个企业技术路线推演，不可能上升到一个新半导体定律。

因为它没有给出足够清楚的时间定义、量化公式和可验证推演。

它表面上说「用时间缩微替代几何缩微」，但最后仍然回到 MTr/mm²，甚至用自家等效公式把 MTr 放大，再去对标 2031 年的 1.4nm / 400+ MTr/mm²。这是口径偷换，不是同尺比较。

LogicFolding 也没有脱离行业既有框架。它本质上仍然是先进封装、3D stacking、SRAM-on-logic、互连缩短、资料流优化、软硬协同这些既有方向的组合。

Samsung 早就展示过 SRAM 叠 logic，AMD 早就量产 3D V-Cache，TSMC SoIC 也早就把异质垂直整合变成成熟平台。华为真正做的是把这些高成本技术拿来补自己的制程短板，而不是发明了一条完全脱离行业框架的新路。

Kirin 9050 把 3D SRAM 做进手机 SoC，这在产品层面上确实是世界唯一。

昨天一大堆小作文把这个堆叠有别于台积电现行方案，甚至还跑出了这是从底层设计上，从前段设计就开始不同，这个堆叠甚至是逻辑门级别的堆叠，类似于GAA下一代CFET结构，这种小作文写得头头是道，满满的专业术语，特别唬人。

但是逻辑门堆叠靠真正最尖端的前段制程，后段的logicfolding是um级别能处理nm级别的加工？这不是开国际玩笑吗？

还有芯片高速信号部分的金属互联单独分到第二片晶圆上，金属互联跟高速信号扯再一起是什么东东？还单独分到第二片晶圆上？这些小作文麻烦大家先用豆包确认一下，都是连基础常识都没有的专业名词东拼西凑。

反正昨天群里无数的华为韬定律跟世界不一样的一大堆说法，我实在不太想再回。

今天我根据华为公布的所有已知数据，去不断收敛，思考Kirin 9050可能的设计与布局，会发现Kirin将是一颗性能不错的芯片，它不是全球第一颗采用3D SRAM的芯片，但百分百肯定是全球第一颗3D SRAM的手机SoC。这是一个产品应用的突破，且独一无二，直得大书特书，但整个过程并没有出现新的技术原理。

别人不是不知道 3D SRAM 有用，而是手机上用它很贵、良率不会太高、收益不一定覆盖成本。华为是在 SMIC 制程落后太多的情况下，被迫采用这种高成本、高风险方案去追 sustained performance。

但好在我们中国的半导体行业一直不太在意成本，因为半导体制造的成本最大头『设备』，一直有国家与政府支持，设备这一个最大的生产成本负担大大减少，让中国所有半导体工厂的低良率或高成本方案变得可实现，这也是中国半导体行业制程虽然有代差，但也是来势汹汹，追赶快速的的一个原因。

这颗芯片在外部散热上也得下不少工夫，这刚好也符合去年Mate 80的kirin 9030推出时，一直传出华为的内置微型主动散热风散＋微泵液冷的双轨主动散热技术。

这是华为从2024年就开始多次公开的技术，非常有可能搭配9050推出，因为这颗采用logicfolding逻辑折叠技术的芯片急需强大的散热能力。

所以除了堆叠的成本，散热装置的成本也不可小觑，这正是这个看起来提升芯片性能不错的原因，也是全球其他竞争者不会采用的主要原因，半导体行业规律一直是追求平等能效下得最低成本方案，高成本方案往往会被淘汰。

很多人对华为有坚定不移的高道德看法，但我必须说一下，我们看产业先把那些道德，家国情仇等等放一边，因为把这些加进来你什么都分析不了，咱们先客观的把事情分析完了，你再加入自已的立场，做分析的时候不要把立场跟事实混合一谈，一定要分开看。

**

### **韬定律最大的问题就是，它叫「时间缩微」，但最后仍然回到 MTr/mm²**

华为对韬定律的官方描述，是用时间常数 τ 的缩短来替代传统几何尺寸缩微，并把这套方法延伸到器件、电路、芯片、系统四个层级。其中 LogicFolding 逻辑折叠技术被描述为能缩短关键路径布线、降低互连电阻与寄生电容，从而提升晶体管密度、性能与能效。

这套说法最大的问题在于，它名义上是「时间缩微」，但公开材料里没有给出足以工程验证的时间定义。

如果 τ 是时间，那它到底是哪一个时间？是 transistor RC delay？是 FO4 delay？是 critical path delay？是 SRAM access latency？是 NoC latency？是 CPU pipeline stage time？是 GPU memory fetch time？还是 app end-to-end execution time？

华为官方把 device、circuit、chip、system 都放进同一个 τ 框架里，听起来宏大，但其实把很多不同层级的问题混在了一起。

device 层的 τ 可以是物理 RC，circuit 层可以是 critical path，chip 层变成 workload-driven instruction/data flow，system 层又变成 UnifiedBus 与 memory semantics。

这些东西都跟「时间」有关，但它们不是同一个可以直接相加、比较、预测的物理量。所以韬定律目前更像是一个设计哲学或工程叙事，跟「定律」一毛钱关系也没有。

真正的定律至少要有变量定义、量测方法、缩放关系、预测公式，以及可验证或可证伪的边界。

但从目前公开内容看，它没有给出「τ 怎么算」「不同层级如何归一化」「性能、能效、密度之间如何量化转换」这些核心定义。

更讽刺的是，何庭波论文说韬定律要抛开几何缩微、改谈时间缩微，但最后宣传口径又回到了 MTr/mm²。

大肆宣传 Kirin 2026 被外部整理为 238 MTr/mm²、晶体管密度提升 53.5%，2031 年目标则是 400+ MTr/mm²、等效 1.4nm。

这就是问题所在，既然核心是时间，为什么最后还是拿晶体管密度去对标台积电未来 1.4nm？

如果真的是一条新的「时间缩微定律」，就应该用 delay、latency、energy-delay product、data movement time、system execution time 这些时间相关指标建立可比性。

但现在的结果是，先宣称「不再看几何」，再用自家公式把 MTr/mm² 放大，最后拿这个「等效密度」去对标业界先进制程的真实物理密度。这就是典型的口径切换。

这个现象我没破口大骂已经是足够收敛了。

### **用自定义 MTr 放大值，去对标业界物理 MTr**

我们前面讨论的口径是：155 MTr/mm² + 83 MTr/mm² = 238 MTr/mm²。这是 LogicFolding 对外最容易被传播的数字。但若按照韬公式修正系数 1.357，这三个数字的工程含义会完全不同。

问题非常清楚，如果 238 是经过韬公式放大后的等效 MTr，那它不能和台积电、三星、Intel 那种物理制程口径下的 MTr/mm² 直接对比。

因为台积电 N3、N2、未来 A14 / 1.4nm 的 MTr/mm²，通常是基于实际制程设计规则、标准 cell、SRAM、logic density、CPP、MMP 等物理微缩能力。

而华为这个 238 或1.4nm的 400+，混入了 3D stacking、投影面积折算、架构等效密度、甚至系统层面的资料流收益。这两者不是同一把尺。

因此，把「2031 年达到等效 1.4nm 密度」拿来对标全球先进制程，本质上是断章取义。

这不是「SMIC 2031 年会做出物理 1.4nm」，而是「华为希望用自己的等效密度口径，让系统结果接近 1.4nm 级」。这两件事差很多。

### **韬定律是把业界已有路线重新包装**

韬定律所涵盖的方向，其实都是业界早就在做的内容，缩短互连、降低 RC、3D stacking、先进封装、chiplet / heterogeneous integration、大 cache / system cache、软硬协同、资料流优化、memory hierarchy 重构、压缩与稀疏计算、OS / compiler / runtime 参与调度。

这些方向都有价值，但它们不构成一个全新的半导体物理定律。

TSMC SoIC 本来就支援不同尺寸、不同功能、不同节点的 known-good dies 做高密度垂直整合，目标就是降低 RLC、提高性能、降低功耗。

Samsung 早在 X-Cube 技术展示中，就用 7nm test chip 把 SRAM 叠在 logic die 上，透过 TSV 缩短 die 间路径，提高资料速度与能效。AMD 3D V-Cache 则早已经量产，官方明确列出 64MB L3 cache die、TSV、direct copper-to-copper bond 等技术。

所以严格说，韬定律没有脱离目前半导体行业框架。

它不是一条新路，而是把先进封装、3D SRAM、资料流优化、系统协同这些已存在的路线包装成一套新的叙事。

真正值得讨论的，不是「韬定律是不是取代摩尔定律」，而是在 SMIC 制程落后的情况下，华为能不能用高成本 3D SRAM 和架构协同，把手机 SoC 的有效性能拉回主流区间。

### **Kirin 9030 的现实：SMIC N+3 仍然落后**

Kirin 2026 / 9050 的设计必须从 Kirin 9030 的现实出发。

而Kirin 9030 采用 SMIC N+3 制程，是 N+2 / 7nm-class 的 scaled evolution。虽然 N+3 有密度进步，但在微缩程度上仍然明显落后于台积电、三星的商用 5nm 先进节点。

这意味着华为不可能只靠制程去追 Qualcomm、Apple、MediaTek。

如果 Kirin 9030 约 120 多 mm²，而且下层 155 MTr/mm² 需要除以 1.357，那么它的真实密度大约是 114 MTr/mm²。

这与我们讨论中的 SMIC N+3 真实 MTr 约 110–120 MTr/mm² 是一致的。

因此 Kirin 2026 / 9050 真正能做的，不是让 N+3 突然变成 N3P，而是用非常昂贵、较为激进的封装和 SRAM 设计去补短板。

### **kirin 2026/9050 的重点不是 CPU，而是 GPU / NPU / 资料搬移**

根据华为官方提升图，以 2025 为基准，2026 年 LogicFolding 的提升大约如下。

![](images/ddbcb930049656b2e6b3.png)

这组数字可能有企业自己的口径与优化，但也提供了一个直观可参考的范围。

2026年的CPU 提升相对保守，GPU 提升大，NPU 提升极大。

这说明 Kirin 9050 不是一颗主打CPU性能的芯片。

从这个提升幅度以及华为韬定律论文里释放的资料，笔者猜测它更像是，下层扩 GPU / NPU compute，上层用 3D SRAM 减少资料搬移，再用软体、runtime、driver 做场景化调度的一颗芯片。

GPU +38%、能效 +40% 这个方向，尤其像「增加近端 cache / HPM，减少 LPDDR 访问」。

NPU +140%、能效 +81% 则不可能只靠 SRAM，必须来自 NPU MAC 阵列扩大、低精度、稀疏、operator fusion、activation compression、compiler / runtime 分块，以及上层 SRAM 作 activation staging。

### **Kirin 2026 / 9050的设计可能下层算，上层存**

根据所有官方公开资料，所有条件不断收敛后，我会这样设计 Kirin 2026 / 9050。

下层 compute die 约 120–123 mm²，上层 active SRAM die 约 36–40 mm²。上层不是 logic die，而是 shared 3D SRAM pool。高热 compute 全部留在下层。

**下层 die，跟上一代的9030差不多 120–125 mm²，不缩小。**

我看到有小作文说华为利用logicfolding逻辑折叠可以比上代虽小面积，但是如果要达到华为官方的性能提升水平，那下面这一颗die还是得跟之前的9030接近，而9030比苹果，高通得竞品都来得大一点。

而9050虽然把SRAM移到了上层，但下层的主die还是得保持原本的面积，这也正是GPU能有不错提升，NPU有大提升的核心原因，永远都是面积的问题，区别是你移走了SRAM，多出来的空间可以增加GPU跟NPU。

所以下层的主die，不应该因为 SRAM 搬到上层就缩小到 100mm²或以下。

原因很简单：华为的问题不是成本最小化，而是制程落后。如果把 SRAM 搬走后释放的面积拿来省成本，那性能追不上。如果把释放的面积拿来扩 GPU、NPU、NoC、电源网格，才有可能对应官方图上的 GPU / NPU 提升。

因此笔者根据华为官方公布的提升比例认为 9050 的下层策略应该是：CPU 小改，GPU 做宽，NPU 扩大，频率不硬拉。

![](images/8a12da1ba7cdada3a1df.png)

**上层 die：36–40mm² 的 3D SRAM，而不是越大越好的暴力方案**

上层 83 MTr/mm² 按 1.357 修正后为 61.16 MTr/mm²。如果主要是 6T SRAM，则 61.16 ÷ 6 = 10.19 Mbit/mm²，换成容量约 1.27 MB/mm²。

如果只是要跟上苹果，高通，联发科等国际旗舰手机SoC 的 SRAM 那 10mm² 左右的面积就够了。

但华为官方图的 GPU +38%、NPU +140% 不是小幅 cache 增量能解释的。

反过来，60mm² 上层 die 又太像桌面级 3D cache，不像手机量产 SoC 的保守热设计。

因此最合理的方案是 36–40mm² active SRAM die，约 39–46MB 可用 3D SRAM。

![](images/0d43758d265eb8d32f50.png)

### **上层 SRAM 到底放什么？**

上层不能放 GPU shader，不能放 NPU MAC，不能放 CPU core，也不应该放 GPU L1 / NPU L0 这种最热、最频繁访问的 local SRAM。

![](images/05eba67f4d256fa1032f.png)

 

上层不是 GPU HPM 固定区 + NPU activation 固定区 + ISP 固定区同时满载。上层是一池 shared SRAM，按场景动态分配。

![](images/14493736da28003dac1f.png)

 

对应官方性能图

![](images/2b969ad8dfc88adf9c40.png)

 因此 9050 的真正架构不是「上下层计算」，而是下层算力扩大，上层资料池降低搬运成本。

### **2027 怎么提升？**

2027 的基础假设是：SMIC N+4 真实 MTr 从 N+3 的约 113 提升到约 130。这相当于 130 ÷ 113 - 1 ≈ 15%。

15% 的真实密度提升很重要，但它不能单独解释华为图上 2027 的巨大提升。

![](images/9e07ec2f3727531c0e81.png)

 

所以 2027 不可能只是 N+4 制程升级。合理推演是，N+4 给下层 compute die 约 15% 密度余裕，第二代 LogicFolding 让上层 SRAM 密度同步提高，上层 active die 从 36–40mm² 提高到 40–44mm²，封装热路径更成熟，GPU/NPU 的 dataflow 和 compression 更成熟，HarmonyOS、driver、compiler、NPU runtime 对 3D SRAM 的调度也更成熟。

如果上层 SRAM 密度按 N+4 比例提升，则 61.16 × 130 ÷ 113 ≈ 70.3 MTr/mm²；换算 SRAM 约 70.3 ÷ 6 ÷ 8 ≈ 1.46 MB/mm²。

这样 2027 的提升才比较合理，不是 N+4 单独救场，而是 N+4 + 第二代 3D SRAM + 更成熟资料流协同共同作用。

### **最终定论：韬定律是叙事，3D SRAM 是补课**

韬定律的时间概念本身就是一个幌子，核心内容只是一个企业技术路线推演，不可能上升到一个新半导体定律。

因为它没有给出足够清楚的时间定义、量化公式和可验证推演。

它表面上说「用时间缩微替代几何缩微」，但最后仍然回到 MTr/mm²，甚至用自家等效公式把 MTr 放大，再去对标 2031 年的 1.4nm / 400+ MTr/mm²。这是口径偷换，不是同尺比较。

LogicFolding 也没有脱离行业既有框架。它本质上仍然是先进封装、3D stacking、SRAM-on-logic、互连缩短、资料流优化、软硬协同这些既有方向的组合。

Samsung 早就展示过 SRAM 叠 logic，AMD 早就量产 3D V-Cache，TSMC SoIC 也早就把异质垂直整合变成成熟平台。华为真正做的是把这些高成本技术拿来补自己的制程短板，而不是发明了一条完全脱离行业框架的新路。

Kirin 9050 把 3D SRAM 做进手机 SoC，这在产品层面上确实是世界唯一。

昨天一大堆小作文把这个堆叠有别于台积电现行方案，甚至还跑出了这是从底层设计上，从前段设计就开始不同，这个堆叠甚至是逻辑门级别的堆叠，类似于GAA下一代CFET结构，这种小作文写得头头是道，满满的专业术语，特别唬人。

但是逻辑门堆叠靠真正最尖端的前段制程，后段的logicfolding是um级别能处理nm级别的加工？这不是开国际玩笑吗？

还有芯片高速信号部分的金属互联单独分到第二片晶圆上，金属互联跟高速信号扯再一起是什么东东？还单独分到第二片晶圆上？这些小作文麻烦大家先用豆包确认一下，都是连基础常识都没有的专业名词东拼西凑。

反正昨天群里无数的华为韬定律跟世界不一样的一大堆说法，我实在不太想再回。

今天我根据华为公布的所有已知数据，去不断收敛，思考Kirin 9050可能的设计与布局，会发现Kirin将是一颗性能不错的芯片，它不是全球第一颗采用3D SRAM的芯片，但百分百肯定是全球第一颗3D SRAM的手机SoC。这是一个产品应用的突破，且独一无二，直得大书特书，但整个过程并没有出现新的技术原理。

别人不是不知道 3D SRAM 有用，而是手机上用它很贵、良率不会太高、收益不一定覆盖成本。华为是在 SMIC 制程落后太多的情况下，被迫采用这种高成本、高风险方案去追 sustained performance。

但好在我们中国的半导体行业一直不太在意成本，因为半导体制造的成本最大头『设备』，一直有国家与政府支持，设备这一个最大的生产成本负担大大减少，让中国所有半导体工厂的低良率或高成本方案变得可实现，这也是中国半导体行业制程虽然有代差，但也是来势汹汹，追赶快速的的一个原因。

这颗芯片在外部散热上也得下不少工夫，这刚好也符合去年Mate 80的kirin 9030推出时，一直传出华为的内置微型主动散热风散＋微泵液冷的双轨主动散热技术。

这是华为从2024年就开始多次公开的技术，非常有可能搭配9050推出，因为这颗采用logicfolding逻辑折叠技术的芯片急需强大的散热能力。

所以除了堆叠的成本，散热装置的成本也不可小觑，这正是这个看起来提升芯片性能不错的原因，也是全球其他竞争者不会采用的主要原因，半导体行业规律一直是追求平等能效下得最低成本方案，高成本方案往往会被淘汰。

![](images/14960407795e0af7dceb.png)

### **对于韬定律一句话总结，『韬定律是包装，LogicFolding 是工程整合，3D SRAM 是高成本补课。』**

韬定律可能让 Kirin 9050 在手机场景里追近主流旗舰，今天我自己去试着大概设计与布局这颗芯片，发现华为在kirin 9050上确实会有令人亮眼的表现，尤其在NPU与AI上。

未来发布之时，铺天盖地的新技术噱头与真实性能提升都会有，说句实话，确实是值得期待，不可否认这是一颗好芯片，敢于不计代价的高水平芯片。

但我们同时也得明白，高成本方案是不得已而为之，他并不是最好最优的技术路线，它更不是新物理，也不是制程有拉近的趋势，自然也不是用一个新名词就能把自定义 MTr 当成等效1.4nm的真实密度来比较，然后去说台积电的也是等效，我为啥不能等效。

华为这条高成本方案正是未来中国特色半导体之路，核心还是靠先进封装这个多年的老生常谈，多了一个韬定律的包装而已。

kirin 9050的真正意义是这个靠先进封装的中国特色半导体之路，由kirin 9050的真正落地开启，这一条高成本方案正常来说没有竞争力应该会被淘汰，但是国内半导体产业没有其他方法，只能死磕。

最后再加上国家的分摊与补贴，或许未来在先进封装上，我们靠着更多的应用，会越做越好，成本越来越低，最终在先进封装上反而有高度竞争力，这一点是华为9050真正落地的最大意义。

而一切的一切都还是得基于科学，基于正常的常识与逻辑，不会也不应该有其他。
