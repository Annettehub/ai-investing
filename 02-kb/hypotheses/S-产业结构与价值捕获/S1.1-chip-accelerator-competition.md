# S1.1 芯片与加速器竞争结构

- **状态**: 活跃
- **旧编号**: H2.1 / H2.3 / H5.2
- **层级**: S 产业结构与价值捕获
- **certainty%**: 55%
- **投资方向**: 纯结构判断，不绑定仓位
- **最后更新**: 2026-07-15

## 核心命题

AI 加速器的价值分配不是“GPU 或 ASIC 二选一”，而是由训练、推理、软件生态、客户定制能力和先进封装供给共同决定。

当前判断：

1. **训练和通用开发仍由 GPU 主导**：CUDA、NVLink、软件栈、开发者生态和系统交付能力构成高切换成本。
2. **推理和稳定大规模负载会向 ASIC 渗透**：当模型架构、推理负载和部署规模足够稳定，云厂商更有动力用自研或定制 ASIC 降低成本。
3. **芯片市场集中度仍高，但价值捕获方式在变化**：NVIDIA 从单卡优势延伸到网络、系统、机架和软件；Broadcom/Marvell 等通过 ASIC 设计服务切入超大客户；AMD、国产芯片和新创公司需要同时突破性能、生态、供给和客户信任。

## 合并后的子问题

| 子问题 | 原内容来源 | 当前判断重点 |
|---|---|---|
| GPU 训练护城河是否被 ASIC 侵蚀 | GPU 训练护城河 | 训练场景仍看 GPU 生态和系统交付，ASIC 更可能先从推理和内部稳定负载切入 |
| AI 芯片集中度是否下降 | AI 芯片集中度 | 新进入者需要同时拿到客户、软件、先进制程、封装和供货能力，集中度下降会很慢 |
| 推理市场 GPU/ASIC 如何分份额 | 推理份额判断 | 推理成本曲线、模型稳定度、客户自研能力和 CoWoS 分配会决定 ASIC 渗透速度 |

## 判断链条

```text
模型训练和迭代频率
-> 软件生态与硬件可替代性
-> 云厂商是否有稳定大规模推理负载
-> ASIC 是否能拿到先进制程、HBM 和先进封装
-> GPU / ASIC / 国产卡的份额与利润分配
```

**更新后的判断**：S1 的核心不再是“GPU vs ASIC 谁更快”，而是“哪个系统能在模型变化、推理成本、供应链约束和客户负载变化中保持最低总成本和最高可交付性”。NVIDIA 当前仍有系统级优势；Google TPU、Amazon Trainium、Meta MTIA 等自研 ASIC 的威胁主要来自 captive customer 和内部规模化负载。

## 复核清单

1. 云厂商 CAPEX 是否继续转向自研 ASIC，还是仍主要流向 NVIDIA 系统。
2. Broadcom/Marvell 的 AI ASIC 收入是否连续兑现，并且毛利不被定制服务稀释。
3. NVIDIA 推理系统的 TCO 是否仍压制 ASIC 替代。
4. 国产 AI 芯片是否只是政策配套采购，还是进入真实高利用率生产负载。
5. CoWoS/HBM 分配是否成为 GPU 与 ASIC 份额变化的领先指标。
