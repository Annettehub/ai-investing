# G3 AI 推理需求规模验证

- **状态**: 活跃
- **旧编号**: H6.2（原"AI 应用与制造需求"已拆分为 G3a/G3b 后废弃，G3b 制造业不追踪，本文件为重新构思后的 G3 最终版）
- **层级**: G 需求与周期
- **产业范围**: L3 基础设施、L4 模型
- **certainty%**: 40%
- **投资方向**: 纯置信度判断，不绑定仓位
- **最后更新**: 2026-07-21

## 核心命题

AI 需求是否正从"训练单一驱动"转向"训练 + 推理双轮驱动"，推理是否成为独立于训练的增量需求来源。

G1 已验证 CAPEX 总量在增长（蛋糕在变大），G2 已验证存储出现分层周期（蛋糕内部结构在分化）。G3 追问的是**需求结构**：增长的 CAPEX 中，有多少比例正在从"训练大模型"转向"运行推理服务"？推理需求的增长是训练需求的附属品（训练完后顺便做推理），还是正在形成自己独立的增速曲线？

**一句话记忆**：G1 问"买不买"，G3 问"买了之后用不用、用多少"。

## 传导链

```text
模型能力达到"够用"阈值（上下文窗口、推理成本下降、多模态成熟）
  -> 应用层出现可规模化的推理产品形态（API 服务、Agent、Copilot、企业工作流）
  -> 推理调用量 / token 消耗 / 企业用户数增长（G3 观测到此为止）
  -> 推理芯片 / 服务器 / 存储 / 网络配套需求增长（G3 观测到此为止）
  -> 是否可持续盈利、ROI 是否为正（→ 交给 R2）
  -> 价值留在哪一层（→ 交给 S1/S3）
```

G3 只追踪前半段：从模型能力到推理使用量再到基础设施配套需求的**扩散速度和结构变化**。不触碰后半段的财务回报和价值分配问题。

## 三层前置指标体系

### 第一层：供给侧结构变化（最前置，6-12 个月领先）

> 这一层追踪芯片厂商和云厂商的供给侧信号。台积电排产、芯片型号出货结构、CSP 采购配置是推理需求最硬的领先指标——比 API 调用量更早 6-12 个月反映需求结构转向。

| 指标 | 为什么前置 | 数据来源 | 当前基准 | 需新增追踪 |
|------|-----------|---------|---------|-----------|
| CoWoS 配额中推理芯片（ASIC + 推理 GPU）占比 | 台积电排产是需求的硬领先指标；推理芯片 die size 通常小于训练芯片，同样的 CoWoS 产能可以封装更多推理芯片 | 吴梓豪模型 / 台积电法说会 / 供应链调研 | G1 已有 ASIC/GPGPU 总量比例（2026 约 50%，2027 约 58%），需拆分为训练用 vs 推理用 | 推理 / 训练分类口径需与吴梓豪模型对齐 |
| NVIDIA 数据中心收入中推理 mix | NVIDIA 产品组合变化直接反映需求结构；从 H100/B200 训练为主到 L40S/RTX 推理 SKU 占比提升是明确信号 | NVIDIA 财报 / IR / 业绩会 | G1 已追踪数据中心收入总量，需拆 inference revenue mix | 需建立 inference mix 的季度追踪 |
| 推理专用服务器（L40S、A10G、RTX A6000 Ada、T4 等）出货量增速 | 推理芯片与训练芯片的出货比例变化是需求结构转向的直接证据 | ODM 出货数据（广达、纬创、英业达）/ 渠道调研 | 无基准，需从零建立 | 新增追踪 |
| 边缘 AI 芯片（Jetson、Movidius、昇腾 310、RK3588 等）出货量增速 | 推理需求扩散到边缘是独立于云端推理的增量信号 | 供应商出货数据 / 行业报告 | 无基准，需从零建立 | 新增追踪 |
| CSP 采购配置中推理优化 SKU 占比 | 云厂商内部训练集群 vs 推理集群的采购结构变化 | 供应链调研 / 吴梓豪模型 | 无基准，需从零建立 | 新增追踪 |

**关键洞察**：G1 中已追踪 NVIDIA 66 万片 CoWoS 配额和 ASIC 37 万片配额。G3 不重复追踪总量，而是追问**这些配额中推理场景的比例**。从芯片型号（H100 训练 vs L40S 推理）、客户部署方式（集群规模 / 网络拓扑）和利用率模式（连续满载 vs 弹性调度）三个角度推断。

### 第二层：需求侧直接信号（同步或略滞后，3-6 个月）

> 这一层追踪模型公司和云厂商的下游使用量数据。推理需求是否从"训练需求的附属品"变成"独立增长极"，关键看推理相关指标的**增速是否持续高于**训练相关指标。

| 指标 | 为什么可衡量 | 数据来源 | 当前基准 | 需新增追踪 |
|------|-------------|---------|---------|-----------|
| Anthropic API 调用量 / Claude 消息量增速 | Anthropic 已披露 Claude API 调用量翻倍节奏；推理-native 公司，几乎全部收入来自推理 | Anthropic 博客 / IR / 第三方监测 | 无基准，需从零建立 | 新增追踪 |
| OpenAI API 调用量 / ChatGPT 活跃会话数 | OpenAI 部分披露 usage 数据；GPT 系列从训练转向推理服务的进度 | OpenAI 博客 / 第三方（Similarweb、Sensor Tower） | 无基准，需从零建立 | 新增追踪 |
| Azure OpenAI Service / Google Cloud Vertex AI 外部 token 收入增速 | 云厂商推理服务收入是推理需求规模化的直接信号 | 微软 / 谷歌财报分项 | G1 不追踪推理收入分项 | 新增追踪 |
| 推理收入占总 AI 收入比例（行业级） | 拆分训练集群一次性销售 vs 推理服务持续性收入 | 行业报告 / 公司财报拆分 | 无基准，需从零建立 | 新增追踪 |
| 微软 Copilot / 腾讯 CodeBuddy / 企业 Agent 活跃用户数 | 企业级推理应用的规模信号 | 公司披露 / IR | 微软已披露 Copilot 企业客户数；腾讯 CodeBuddy 待追踪 | 新增追踪 |

**关键洞察**：这些指标看**增速不看绝对值**。推理需求是否成为独立增长极，取决于推理相关指标的增速曲线是否持续高于训练相关指标。如果两者增速趋同，说明推理仍是训练的附属品；如果推理增速显著更高且持续多个季度，说明独立的推理需求曲线正在形成。

### 第三层：基础设施配套信号（与 G2 联动，验证推理需求的真实性）

> 这一层是 G3 与 G2 的交叉验证点。推理需求增长必然拉动特定基础设施配套，如果这些配套不动，则推理需求信号存疑。

| 指标 | 为什么与推理相关 | 数据来源 | 与 G2 的联动 | 需新增追踪 |
|------|----------------|---------|------------|-----------|
| 企业 SSD / AI SSD 订单量增速 | 推理需要 KV cache 存储、向量数据库、RAG 检索存储、低延迟读取 | TrendForce / 供应商法说会 | G2 第三层"成长性验证中"——推理增长是 eSSD 核心需求驱动之一 | G2 已追踪，G3 追问"eSSD 增长是否由推理拉动" |
| 低延迟网络设备（RDMA、InfiniBand、RoCE v2）在推理场景出货占比 | 推理对响应延迟敏感，需要不同于训练的网络架构；训练用大规模 RDMA 集群，推理用低延迟小集群 | 供应商出货 / 吴梓豪模型 | G1 已有网络设备总量追踪 | 需拆推理场景占比 |
| 内存带宽型服务器配置占比（HBM3e / LPDDR5X / CXL 内存扩展） | 长上下文推理对内存带宽需求高于训练对算力密度需求 | ODM 配置数据 / 供应商 | G2 已追踪 DDR5 / HBM | 需按推理场景拆分 |
| 推理优化型数据中心功率密度变化（W/rack）与部署节奏 | 推理服务器功耗特征（较低功率、弹性调度）不同于训练集群（高功率、连续满载） | 数据中心运营数据 / CDW 等渠道 | G1 已追踪数据中心容量 | 需按工作负载类型拆分 |

**关键洞察**：这一层是 G3 → G2 的反馈通道。如果第二层显示推理 token 快速增长，但第三层 eSSD/网络/内存配套不动，有两种可能：(a) 推理需求信号是假的（token 增长但未形成真实基础设施需求），或 (b) 供给端有瓶颈（eSSD 良率、网络设备交付周期）。这个分歧需要回写到 G2 的"保留分歧"中。

## Kill Metrics

> 证伪条件必须可观测、有具体阈值和时间窗口。触发任一条件 → certainty% 归零，移入 false-beliefs。

| 证伪条件 | 阈值 | 时间窗口 | 当前状态 |
|---------|------|---------|---------|
| 主要推理平台（Anthropic / OpenAI / Azure OpenAI）API 调用量或 token 消耗连续 2 季度环比零增长 | 环比增速 ≤ 0% | 2 季 | 待建立基准 |
| 推理收入占总 AI 收入比例连续 2 季度下降 | 占比连续下降 | 2 季 | 待建立基准 |
| NVIDIA 推理专用芯片（L40S / A10G / RTX 推理 SKU）出货量连续 2 季度环比下降 | 环比降幅 > 15% | 2 季 | 待建立基准 |
| 推理 token 增速与 eSSD 订单增速脱钩 | token 环比增速 > 20% 但 eSSD 订单环比增速 < 5%，持续 2 季 | 2 季 | 待建立基准 |
| 云厂商明确下修推理相关 CAPEX 指引 | 任一 CSP 在财报或法说会中明确将 CAPEX 下调归因于"推理需求低于预期" | 任一季度 | 待观测 |

**与 G1 Kill Metric 的区别**：G1 看总量（CAPEX 下修、利用率恶化），G3 看结构（推理占比下降、推理 SKU 出货下滑、使用量停滞）。G1 的 CAPEX 下修可能是训练需求放缓，不一定是推理需求出问题；G3 专门捕捉推理需求的独立信号。

## 与 G1 / G2 / S / R 的边界声明

| 信息 | 归属 | 原因 | G3 如何处理 |
|------|------|------|------------|
| 训练 + 推理芯片总出货量、CoWoS 总配额 | **G1** | 总量验证 | 引用 G1 数据，不重复追踪 |
| CoWoS / 出货量中推理 vs 训练的占比 | **G3** | 需求结构变化 | 核心追踪指标 |
| GPU vs ASIC 在推理中的份额变化 | **S1** | 技术路线竞争格局 | 如果推理增长确认，S1 追问"谁拿到份额"；G3 只问"推理是否在增长" |
| 推理芯片定价权和毛利率 | **S1** | 价值捕获 | G3 不触碰 |
| HBM / DDR5 / eSSD 总量和周期 | **G2** | 存储分层周期 | 引用 G2 数据；G3 追问"eSSD 增长是否由推理拉动"并反馈给 G2 |
| 推理收入是否转化为供应商利润和现金流 | **R1** | 上游财务兑现 | G3 不触碰 |
| 推理 ROI 是否为正、客户是否持续付费 | **R2** | 下游可持续性 | G3 不触碰 |

**硬边界**：G3 的证据时间线中如果出现"毛利率""NRR""ROI""客户续约率""FCF"等关键词，说明越界了，应移交给 R1/R2。

## 关联关系

- **依赖 G1**: G3 假设 G1 的 CAPEX 总量增长为真，追问的是总量中的结构变化
- **联动 G2**: G3 的推理需求增长是 G2 第三层（eSSD / AI SSD）"成长性验证中"的关键需求端验证；G3 向 G2 输出"推理→eSSD"传导信号，G2 向 G3 反馈供给端瓶颈
- **输出给 S1**: 若 G3 确认推理需求独立增长，S1 追问 GPU vs ASIC 在推理中的份额竞争
- **输出给 R1/R2**: 若 G3 确认推理使用量规模化，R1 追问推理收入是否转化为供应商利润，R2 追问客户 ROI 和续费
- **关联 company**: NVDA（推理 SKU 出货结构）、Anthropic（API 调用量）、OpenAI（API + ChatGPT 使用量）、Microsoft（Copilot + Azure OpenAI）、Google（Vertex AI）、TSM（CoWoS 推理配额）
- **关联 theme**: AI 推理、训练-推理结构转换、token 经济

## 复核条件

> 每月 Kill Thesis 回看时，同时检查以下问题：

1. 当前证据是否支持"推理正在形成独立于训练的增速曲线"？最核心的一条证据是什么？
2. 第一层（供给侧结构）和第二层（需求侧使用量）的信号是否方向一致？如果矛盾，哪个更可信？
3. 第三层（基础设施配套）是否与第二层联动？如果推理 token 增长但 eSSD/网络配套不动，原因是什么？
4. 是否有新出现的反方证据（如"推理需求只是训练需求的滞后附属品"）未被纳入？
5. 是否有证据越界进入了 S 层（份额竞争）或 R 层（财务回报）？是否需要移交？
