# Hypotheses 假设判断层

> 假设层不是资料库，也不是概念百科。它把已经整理的事实转化为**可跟踪、可证伪、能进入投资判断的因果命题**。

## 一、它在知识循环中的功能

```text
03-raw 原始资料
  -> sources 来源卡：这条信息说了什么，可信度多高
  -> concepts 概念层：技术、产品和产业链事实是什么
  -> hypotheses 假设层：这些事实会如何改变需求、价值分配和业绩
  -> reports / tracking / dashboard：形成研究输出并持续验证
  -> 新信息再次回写 certainty、触发条件和反证
```

假设层承担三个任务：

1. **压缩信息**：同一主题的多篇资料，最终只更新少数关键命题。
2. **连接研究与决策**：把技术事实推导成需求、竞争格局、利润流向和财务兑现。
3. **防止确认偏误**：每条假设必须有反证、阈值、时间窗口和置信度变更记录。

一句话记忆：**概念层回答“是什么”，假设层回答“所以会怎样，以及什么情况下我错了”。**

## 二、GSR 三段因果链

旧版 D1-D7 把趋势、需求、技术、壁垒、产业链和公司基本面并列，容易让同一条因果链散落在多个维度。现在统一为三个连续问题：

> **G：蛋糕是否变大？**<br>
> **S：蛋糕如何切，谁有权多拿？**<br>
> **R：拿到的蛋糕是否真正进入财务报表？**

| 层 | 名称 | 核心问题 | 主要观测对象 | 合并的旧维度 |
|---|---|---|---|---|
| **G** | 需求与周期 Growth | 需求是否增长、供需缺口是否持续、周期处于什么位置 | CAPEX、出货、库存、价格、产能、渗透率 | D1 Trend + D6 Demand |
| **S** | 产业结构与价值捕获 Structure | 哪条路线胜出、瓶颈在哪里、谁有壁垒、利润流向谁 | 技术路线、竞争格局、份额、定价权、产业链价值量 | D2 Moat + D5 Tech + D7 Chain |
| **R** | 业绩兑现 Realization | 产业优势是否转化为公司收入、利润、现金流和合理估值 | 营收、毛利率、订单、自由现金流、资本回报 | D3 Fund |

宏观、政策和地缘政治不再单列一层，而作为 `geo`、`policy`、`macro` 等**跨层风险标签**附着在 G、S 或 R 命题上。

产业位置也不再与 GSR 混用。每条假设另加范围标签：`L1 能源`、`L2 芯片`、`L3 基础设施`、`L4 模型`、`L5 应用`、`X 跨层`。

## 三、S 的准确含义

这里的“结构”是**产业如何组织、竞争以及分配价值的结构**，包括四部分：

1. **技术架构**：GPU、ASIC、存储、互联、封装等路线如何组合，替代边界在哪里。
2. **瓶颈结构**：先进制程、CoWoS、HBM、网络、电力等哪个环节限制系统扩张。
3. **竞争与壁垒结构**：份额集中度、客户切换成本、生态、认证和规模经济决定谁有定价权。
4. **价值分配结构**：客户的一元支出最终由芯片设计、晶圆代工、封装、设备、云和应用分别拿走多少。

### 与 G 和 R 的区别

以台积电为例：

- **G 需求与周期**：AI CAPEX 是否继续增长，先进制程需求是否上升。
- **S 产业结构与价值捕获**：先进制程和 CoWoS 是否构成瓶颈，台积电是否不可替代，能拿走多少产业利润。
- **R 业绩兑现**：上述优势是否转化为台积电收入、毛利率、自由现金流和合理估值。

以 AI 应用为例：

- **G**：Agent、Copilot、企业 AI 的用户量和 Token 需求是否增长。
- **S**：价值最终留在模型、云厂商、应用公司，还是掌握客户入口的平台。
- **R**：应用公司是否形成 ARR、留存率、毛利率和现金流。

### 需要注意的边界

不是所有技术信息都进入 S：

- “HBM4 带宽提高”是技术事实，放在 `concepts`。
- “HBM4 认证和封装能力使头部厂商保持定价权”才是 S 类假设。
- “SK 海力士因此收入和利润超预期”则进入 R。

只有当技术变化会影响**瓶颈、竞争格局、定价权或利润流向**时，才进入 S。

## 五、假设标准模板

```markdown
# {G/S/R编号} [{层名}][{产业范围}] {一句话可证伪判断}

- **状态**: candidate / active / active-tracking / paused / falsified / validated
- **旧编号**: Hx.x（如有）
- **certainty%**: N%
- **最后更新**: YYYY-MM-DD

## 核心命题
用一段话写清“因为 A，所以 B；通过 C 验证”。

## 传导链
事实 -> 需求/结构变化 -> 价值分配 -> 可观测结果

## Kill Metrics
| 证伪条件 | 阈值 | 时间窗口 | 当前状态 |

## 证据时间线
| 日期 | 证据 | 支持/反驳/中性 | 来源标签 |

## 下一步跟踪
| 指标 | 当前值 | 触发阈值 | 频率 | 数据源 |
```

### 强制规则

1. 假设必须可证伪，Kill Metric 必须同时有**指标、阈值和时间窗口**。
2. 初次创建 `certainty <= 60%`；上调必须来自新增的独立证据。
3. 技术事实先进入 concepts，只有形成投资含义后才进入假设。
4. 同一信息可同时影响 G、S、R，但必须分别写清传导关系，不能重复粘贴原文。
5. 超过 35 天未回看，状态标记为过期待审；触发证伪后移入错题本。

## 六、置信度标尺

| 区间 | 级别 | 证据要求 |
|---|---|---|
| 0-29% | 推测 | 单源或直觉，尚无硬数据 |
| 30-49% | 待验证 | 一个可靠来源加一个交叉信号 |
| 50-69% | 可能成立 | 至少两个独立来源，方向一致 |
| 70-89% | 高置信 | 至少三个独立来源，且经历一次成功验证 |
| 90%+ | 基本确认 | 多元验证和公开数据持续确认，可考虑晋升为规则 |

证据标签继续使用：`[本地]`、`[网页]`、`[数据]`、`[知识星球]`、`[推测]`、`[待验证]`。

## 七、生命周期与小循环

```text
候选 -> 审核转正 -> 活跃跟踪 -> 新证据更新 certainty
                         -> 证伪：进入 false-beliefs
                         -> 多轮验证：晋升为 rule
                         -> 研究完成：形成 report + tracker + dashboard
```

一次真正的小循环至少完成：

1. raw 入库并去重；
2. 来源卡提取和信源分级；
3. concepts 更新事实；
4. G/S/R 假设更新判断、反证和 certainty；
5. 输出研究稿或周度复盘；
6. 跟踪表记录下一次需要观察的数据与触发器。
