# R2 下游兑现：最终采用者是否获得可持续回报

- **状态**: active-tracking
- **旧编号**: H7.1；吸收旧 R1.2 中与最终采用者 ROI 有关的内容
- **层级**: R 业绩兑现
- **产业范围**: 模型、云、企业软件、Agent、Copilot、垂直应用及最终采用者
- **certainty%**: 40%
- **最后更新**: 2026-07-21
- **投资方向**: 判断 AI 应用支出是否具备长期支付能力，不直接绑定仓位

## 核心命题

AI 应用只有在最终采用者获得**可量化、可重复、扣除全部成本后仍为正的净收益**时，才完成下游兑现。最终采用者包括购买 AI 服务的企业、机构或个人；“推理”是技术运行过程，不是最终消费者。

当前判断是：AI 已在部分工作流中显著缩短周期、降低迁移或生产成本，但大规模企业采用仍处于早期，Token 使用量和厂商 AI 收入增长尚不能证明最终采用者已经形成稳定 ROI，因此 certainty 暂定为 40%。

## 传导链

```text
模型能力与推理可用性
-> 嵌入真实工作流
-> 用户采用、使用频率与流程重构
-> 增量收入 / 成本节约 / 周期缩短 / 风险下降
-> 扣除模型、推理、集成、数据、治理与组织成本
-> 净 ROI 与回收期
-> 续费、扩容和持续预算
-> 应用/模型/云厂商 ARR、利润和现金流
```

R2 同时检查两本账，但不能混为一谈：

1. **采用者 ROI 账**：客户使用 AI 后是否真的多赚钱、少花钱、缩短周期或降低风险。
2. **供应商变现账**：提供 AI 产品的公司是否形成续费、扩容、毛利和现金流。

供应商 ARR 增长可以来自试用期预算或资本补贴；只有客户自身 ROI 为正，供应商收入才更可能持续。

## 两本账的验证指标

| 账本 | 核心问题 | 主要指标 | 常见误判 |
|---|---|---|---|
| 最终采用者 | AI 是否创造净经济收益 | 增量收入、节省工时、单位成本、周期时间、错误率、回收期、规模化采用率 | Token 多、席位多不等于 ROI 为正 |
| AI 应用/软件供应商 | 客户收益是否转成持续付费 | AI ARR、RPO、NRR、付费席位、使用频率、续费率、ARPU | 签约额不等于收入和现金流 |
| 模型与云供应商 | 推理需求是否具备健康单位经济 | 推理收入、单位 Token 毛利、客户集中度、推理成本下降、现金消耗 | 调用量增长不等于可盈利 |
| 系统集成与咨询 | 企业是否从试点进入生产 | 项目周期、生产部署率、签约转收入、客户复购 | PoC 数量不等于规模化采用 |

## 与 G、S 的边界

| 信息 | 应放层级 | 原因 |
|---|---|---|
| 用户量、Token 量、Agent 使用量增长 | G | 说明需求或活动量变大 |
| 价值留在模型、云、应用还是客户入口 | S | 说明产业如何分配价值 |
| 最终采用者节省成本、增加收入并持续付费 | R2 | 说明采用者获得经济回报 |
| 应用公司形成 ARR、留存、毛利和现金流 | R2 | 说明供应商完成财务兑现 |

旧 R1.2 中关于字节 Token 增长、GPU/ASIC 价值迁移、先进制程和光互联定价权的内容分别属于 G 或 S，不再作为 R2 的直接证据。

## 关联关系

- **需求前提**: G3 AI 应用与制造需求
- **结构前提**: 应用、模型、云与客户入口之间的价值分配属于 S 层，后续形成独立命题时与 R2 联动
- **上游关系**: R1 可以因基础设施建设先行而短期成立；R2 决定这种投入是否有长期支付来源
- **公司验证工具**: R0 公司业绩兑现检查清单
