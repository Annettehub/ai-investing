# L3 基础设施层触发器

> 定位: Infrastructure 是把芯片、网络、服务器、电力和系统软件组织成 AI 工厂的层级。本层不再维护多层级百科，只保留触发器仪表盘；一旦触发，再回到 L2 芯片、L4 模型或跨层动态做深度判断。

## 观察原则

- L3 只记录会改变算力建设节奏、集群效率或供应链利润分配的信号。
- 光模块、CPO、AI 服务器、MLCC、CUDA/NCCL/FlashAttention 等旧主题不再单独铺目录，统一作为触发器和证据项管理。
- 如果某个 L3 信号变成长期研究主线，再迁移到 L2、L4 或跨层动态，而不是在 L3 继续分叉。

## 触发条件

| 触发器 | 观察指标 | 触发含义 | 关联层级 | 当前状态 |
| --- | --- | --- | --- | --- |
| 1.6T 光模块提前量产 | 1.6T 出货、交换机芯片供给、客户验证进度 | AI 集群带宽升级快于预期，光模块链条可能提前兑现 | L2 先进封装、L4 推理需求 | 重点跟踪 |
| CPO / 硅光商业化突破 | 博通/英伟达/台积电 CPO 产品量产、硅光良率、客户导入 | 电互联功耗墙被突破，光模块和先进封装价值重分配 | L2、跨层动态 | 重点跟踪 |
| 云厂商自研光模块量产 | CSP 自研光模块、交换机、网络架构披露 | 供应链议价权可能从独立光模块厂转向云厂商 | L3、L5 成本结构 | 待跟踪 |
| CUDA 兼容性被打破 | ROCm、Triton、XLA、国产生态迁移成本下降 | GPU 软件护城河松动，ASIC/国产芯片替代概率上升 | L2 GPU vs ASIC、L4 模型 | 待跟踪 |
| 新通信协议替代 NCCL | RDMA、以太网、专用互联协议大规模替代 | 集群效率和网络设备价值链变化 | L3、L4 | 待跟踪 |
| 代工地域大规模转移 | AI 服务器、光模块、PCB 产能从中国/台湾向海外迁移 | 关税、合规和供应链安全成为定价变量 | L2 地缘政治、L3 供应链 | 待跟踪 |
| FlashAttention 推理效率提升 > 50% | 推理吞吐、显存占用、KV cache 优化、序列并行改进 | 推理成本曲线下移，影响芯片需求和应用定价 | L4 模型、L5 应用 | 待跟踪 |

## 已合并的旧观察主题

| 原主题 | 保留在本页的核心信息 |
| --- | --- |
| AI 服务器供应链 | L4 烧钱 -> L3 数据中心扩建 -> L2 芯片/被动元件/光模块需求爆发；训练:推理从 8:2 反转为 2:8 是算力紧缺的核心背景 |
| MLCC 周期 | AI 服务器高容 MLCC 用量增加；产能利用率、交期延长、高端特种品涨价是供需紧张信号 |
| 光模块产业链 | 400G -> 800G -> 1.6T 迭代仍是 AI 数据中心带宽升级主线；1.6T 上量受交换机芯片和客户验证约束 |
| CPO / 光互联 / 硅光技术 | SerDes 224G 接近物理极限，CPO 通过缩短电信号距离降低功耗，但标准化、良率和可维护性仍是风险 |
| 系统软件 | CUDA、NCCL、FlashAttention、PagedAttention 等只在改变芯片选择或推理成本时触发深入研究 |

## 触发后处理流程

1. 判断信号影响的是建设节奏、单位成本、供应链利润率还是技术路线。
2. 若影响芯片选择，回看 [[concepts/L2-芯片层（Chips）/技术路线/GPU-vs-ASIC竞争格局|GPU vs ASIC 竞争格局]]。
3. 若影响模型效率，回看 [[concepts/L4-模型层（Models）/技术路线/模型架构演进（Transformer、MoE、推理优化、物理AI）|模型架构演进]]。
4. 若影响商业化成本，回看 [[concepts/跨层-方法论（Cross-Layer）/跨层动态跟踪|跨层动态跟踪]]。
5. 若影响投资假设，更新 hypotheses、entities 和当周复盘。
